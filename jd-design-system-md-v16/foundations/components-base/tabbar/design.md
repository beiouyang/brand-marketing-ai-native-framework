## 一句话定义

底部导航栏(Tabbar)位于屏幕底部,**作为页面导航的最高层级**,管控全局,点击切换整页内容。它分为**常规底导**和 **Joy Agent 组合底导**两种形态,支持 **2~5 个坑位** + 可选**灵动岛运营资源位**,并在不同 OS 端/版本下选择不同**材质**(iOS 26+ 液态玻璃 / Android 与 iOS 老系统毛玻璃)。

## Tabbar 与其它导航的边界

| 维度 | Tabbar(本组件) | NavBar(顶部) | Drawer / 侧抽屉 |
|---|---|---|---|
| 层级 | **一级,全局** | 二级及以下,页面内 | 全局或二级 |
| 是否常驻 | 是 | 否(随页面) | 否(临时召唤) |
| 承载 | Tab 切换 + 灵动岛 + Joy Agent | 返回/标题/操作按钮 | 账户/偏好/长尾入口 |
| Tab 数上限 | **≤ 5**(章节 01 a) | — | — |

当一级功能超过 5 个,应在**内容层**重新组织(分类/分组),而**不是给 Tabbar 加 Tab**。

### 02.1 常规底导基础布局

> 子节点 `312:46904` · 1426×1476。

**总高与安全区(图 1)**

| 项 | 取值 | 备注 |
|---|---|---|
| 底导总高 | **69 DP** | 含 iOS 系统安全区 |
| 导航实际高度 | **52 DP** | 真正可用区 |
| iOS 系统安全区 | **17 DP** | ❌ **不可放置任何操作** |
| 内容距导航内间距 | 4 DP | 顶 |
| 内容高度 | **44 DP** | 即坑位高度 |

**底导外尺寸**

| 项 | 常规底导 | Joy Agent 组合 |
|---|---|---|
| 底导宽度 | **351 DP** | **319 DP**(留 8 DP 给 Agent 间距 + Agent 52 DP) |
| 容器圆角 | `radius_xxl` 16 DP | 同 |
| 内边距(左/右) | 12 DP | 12 DP |
| 容器描边 | 1 px 内描边高光 | 同 |

> **容器描边**:Relay 容器 fill 用了 `background-blend-mode: normal, color-dodge` 双层 —— 第二层 `color-dodge` 是液态玻璃边缘的 hairline 高光,导出时丢层。spec-page.html 暂以 `box-shadow: inset 0 0 0 1px rgba(255,255,255,0.6)` 近似还原。⚠️ **估值待对齐**:`color-dodge` 层的确切色值 / 宽度 / 不透明度需设计师从 Relay 回填。

### 02.2 坑位数 2~5 变体(图 2~5)

> **Relay 双子节实测**:常规底导 `312:46904` 351 DP 宽 / Agent 组合底导 `312:47132` 319 DP 宽,各坑位宽不同。两套数值并列对照:

| 坑位数 | 布局 | 常规底导单坑位宽 ≈<br>(351 DP) | Agent 组合单坑位宽 ≈<br>(319 DP) | 间距 |
|---|---|---|---|---|
| 5 | 均分,自动布局 | **71.8 DP** | **65.4 DP** | −4 DP(重叠) |
| 4 | 均分,自动布局 | **88.75 DP** | **80.75 DP** | 0 |
| 3 | 均分,自动布局 | **117 DP** | **106.33 DP** | 0 |
| 2 | 均分,自动布局 | **173.5 DP** | **157.5 DP** | 0 |

> **均以自动布局计算为准**,上表的「单坑位宽 ≈」是按容器宽度实测,仅供研发对齐。Agent 组合形态下,坑位整体缩窄 ~6~10 DP(因 Agent 占 36 DP + 间距 8 DP)。

### 02.3 灵动岛坑位变体(图 6~7)

> 同样有两套实测:常规底导 vs Agent 组合底导。灵动岛尺寸**保持不变**(131×44 / 144×52),但左右余量不同:

| 形态 | 坑位尺寸 | 距底部安全区 | 常规底导 余量(左 + 灵动岛) | Agent 组合 余量(左 + 灵动岛) | 内部 padding(左/中/右) |
|---|---|---|---|---|---|
| 日常 / 运营灵动岛 | **131×44** DP | 4 DP | 49 + 131 | **41 + 131** | 6 / 6 / 6 DP(A·B 双区) |
| 大促灵动岛 | **144×52** DP | 4 DP | 45.75 + 144 | **37.75 + 144** | 12 / 8 / 10 DP(A 区压缩,B 区留余) |

> 灵动岛**位于第 3 个坑位**(2 坑除外),左侧剩余宽度由「容器宽 − 灵动岛宽 − 右余量」自动计算。

## 章节 04 · 应用场景

> Relay 节点 `312:47792`,1426×3864。本章节由 **18 个真实首页 + Tabbar 样张** + 230+ 子 frame 组成,**无独立 description 文本节点**。下方按「**形态 × 灵动岛 × 招手**」组合自动归纳为 **4 类典型场景**(v0.7 由 outline Q1(b) 自动生成,设计师 review 后可调整)。

### 04.1 场景 A · Joy Agent + 常规底导(无灵动岛 / 无招手)

**这是 Tabbar 的「默认」状态**,占 18 样张中的大多数行(第 1 行第 1-2 列)。

**适用业务**:
- 主功能区一级导航:首页 / 频道页 / 列表页 / 详情页
- 大促前 / 大促结束后的日常状态
- 通用 BU:京东主站 / 京东超市 / 京东到家 等

**视觉特征**:
- 左侧 52×52 Joy Agent 默认形态,抽缩 16 DP
- 右侧 4-5 坑位均分(319 DP 容器),第 1 或第 2 位选中(`jdred` + `gray_6` 底)
- 无招手 / 无灵动岛 / 无营销态

### 04.2 场景 B · Joy Agent + 常规灵动岛

**适用业务**(章节 03.3 常规灵动岛色板 + 商品图):
- 日常**商品复购提醒**:鲜奶定期购到期 / 订阅服务续费提醒
- 业务线:京东超市 / 京东到家 / 京东 PLUS 会员 / 京东养车

**视觉特征**:
- 左侧 Joy Agent **抽缩 28 DP**(为灵动岛让位 12 DP)
- 右侧第 3 位被 **131×44 常规灵动岛** 替换
- 灵动岛内:商品图 32 DP(白底,需 PNG 透明素材)+ 黑字双行(`gray_1` 主 + 灰副)
- 其他坑位**选中态背景消失**(`gray_6` 不渲染)

### 04.3 场景 C · Joy Agent + 运营 / 大促灵动岛

**适用业务**(章节 03 运营 / 大促色板):
- **运营会场推广**:抢 5 折商品 / 限时秒杀 / 京东超市频道日
- **重点大促活动**:618 / 11.11 / 京东国际 / 京东 PLUS 大促节奏
- 业务线随大促节奏切换(预热期运营 → 高潮期大促 → 返场期运营)

**视觉特征**:
- 左侧 Joy Agent 抽缩 28 DP(同 04.2)
- **运营型 131×44**:同频色渐变底(本例绿:抢 5 折商品 / 京东超市)+ 100% 渐变描边
- **大促型 144×52**:异型背板(本例红:领取专属福利 / 抢万元红包 CTA)+ 模糊值 8
- **运营 vs 大促** 视觉差:尺寸 + 描边比例 + 模糊 + 背板形状

### 04.4 场景 D · Joy Agent + 招手气泡

**适用业务**(对话中心入口主动 outreach):
- **新用户 onboarding**:首次进入 APP / Joy AI 首次激活引导
- **长期未使用提醒**:30 天未点击 Agent 入口 / 新功能上线
- **业务场景推荐**:基于用户行为推送的对话能力(如「帮我推荐手机」)

**视觉特征**:
- Joy Agent **默认抽缩 16 DP**(气泡场景下 Agent 不需额外抽缩)
- 上方挂**紫粉渐变气泡**(≤18 汉字)
- 气泡距 Agent 底 8 DP / 距右页边 16 DP / **缓出向上弹出动效**
- 同时存在灵动岛时,**数字 / 文字招手降级红点**(章节 02.5 d)

### 跨场景注意

| 项 | 规则 |
|---|---|
| **场景切换动效** | Agent 抽缩 16 ↔ 28 DP **应有缓动过渡**,非瞬切 |
| **灵动岛 vs 招手共存** | 招手数字 / 文字**自动降级红点**,灵动岛消失后恢复(章节 02.5 d) |
| **大促节奏切换** | 同一 BU 在日常 / 预热 / 高潮 / 返场可切 4 类场景,Tabbar 形态保持不变,**只换灵动岛内容** |
| **常规营销 vs 灵动岛** | 营销态(38×38 圆形图,章节 02.4 c)与灵动岛**互斥**,同一坑位不可叠加 |

### 设计师可深化的方向

> v0.7 自动归纳的 4 类是覆盖 80% 业务的典型场景,以下方向需要设计师人工补充:

- 不同 BU(京东超市 / 京东到家 / 京东健康 等)的**坑位顺序差异**(主流程 vs 二级业务的取舍)
- 大促节奏下**灵动岛切换的时序**(谁先 push 给客户端、降级规则、消失策略)
- Joy Agent 招手的**触发逻辑**(被动 push / 主动 schedule / 用户行为驱动 等)

## 交互行为

- **基础切换**:点击坑位,切换整页内容(章节 01)
- **状态切换**:默认态 ↔ 选中态(图标 + 文本变 `jdred`,背景变 `gray_6`)

- **招手降级**:底部存在灵动岛时,文字 / 数字招手**自动降级为红点型**,灵动岛消失后恢复
- **Joy Agent 招手**:气泡**缓出向上弹出**,最多 18 汉字
- **Joy Agent 点击**:进入对话中心场景弹层
