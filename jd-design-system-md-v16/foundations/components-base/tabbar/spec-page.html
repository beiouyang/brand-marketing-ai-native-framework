1组件定义 / 设计原则

👉 一句话理解:打开 JD App,屏幕最底下永远在那条小条 —— 装着「首页 / 分类 / 购物车 / 消息 / 我的」这种最重要的几个图标 —— 这就是 Tabbar(底部导航栏)。

它解决什么:让用户在 App 里**几个最核心的功能之间一键切换**(无论你逛到第几层、看了什么内容,它永远在那等你回到一级页面)。新版还在它上面集成了 AI 助手入口(Joy Agent)和大促/活动的动态展示位(灵动岛)。

为什么有一套规范:Tabbar 是 App 的"门面" —— 用户每天看几十次,任何不一致(图标大小不一、文字超长、坑位数变来变去)都会让人觉得 App 廉价或迷路。所以 V16 制定了从尺寸、颜色、字号到禁止行为的全套规则。

底部导航栏(Tabbar)是 JD APP 屏幕底部的全局导航组件,作为页面导航的最高层级管控全局,点击切换整页内容。

它的核心边界是「切换整页内容」而非「承载子页面」:Tabbar 内的坑位是一级 Tab,各 Tab 之间无父子关系;二级及以下页面应由顶部 NavBar 或 SegmentControl 承担。当一级功能超过 5 个时,应在内容层重新组织而非给 Tabbar 加 Tab。

维度TabbarNavBar(顶部)Drawer / 侧抽屉
层级一级(全局)二级及以下(页面内)全局或二级
承载交互Tab 切换 + 灵动岛 + Joy Agent返回 / 标题 / 操作按钮账户 / 偏好 / 长尾入口
典型坑位数2 ~ 51 主标题 + ≤ 2 操作无上限
常驻可见是是(页面内)否(收起态默认隐藏)
位置屏幕底部屏幕顶部左 / 右侧

V16 立场:Tabbar 是 APP 唯一全局导航,不允许在底部并行存在其它常驻导航条;不允许超过 5 个一级 Tab(章节 01 建议上限);不允许定制坑位高度或图标尺寸偏离 44×44 / 20×20 规范。

1.0 全局引导说明

章节 01 同层级的导览(Relay 节点 312:46899)给 PM / 设计师提供形态边界的一句话规则:

1.1 行为准则

  1. 建议 2 ~ 5 坑位,超过 5 个时应在内容层重新组织(章节 01 设计原则)
  2. iOS 安全区不可放置任何操作 — 17 DP(章节 02 基础布局 a)
  3. 坑位文本最多 4 个汉字,超出截断或换设计(章节 02 默认态)

2.2 容器与坑位结构

Tabbar 由容器 + N 个坑位(可选 Joy Agent + 可选灵动岛)组成,其中容器和至少 2 个坑位为必要元素。

👉 一句话拆解:Tabbar = 1 个外壳(底部那条)+ N 个坑位(图标按钮)+ 可选 左侧 Agent + 可选 嵌入灵动岛。

每个坑位是个 4 件套:

  • ① 图标(20×20)
  • ② 文字标签(最多 4 个汉字,超了截断)
  • ③ 三态切换(默认灰 / 选中红 / 营销态可换图)
  • ④ 可选招手位(右上角红点 / 数字角标 / 短文字气泡)

Joy Agent 是什么? 一个 52×52 的圆形 AI 助手按钮,固定在 Tabbar 左侧。点击进入对话弹层,头顶可以飘"问问我大促攻略"那种气泡(最多 18 个汉字)。

灵动岛是什么? 嵌在「消息」坑位上的一条宽资源展示位,3 种形态(常规商品 / 运营 / 大促),都是 131-144 宽的圆角矩形。展开时占用消息位,所以同时只能展示一条。

3灵动岛运营资源位

灵动岛是嵌入 Tabbar 消息坑位的运营展示位,3 种形态承载日常 / 运营 / 大促不同诉求。同时只能展示 1 条,展开期间消息坑位选中态背景消失。

👉 想插一条运营资源位 / 大促展示位吗?

  • 纯日常商品推荐 → 常规型灵动岛(占消息位,纯色背景,简洁)
  • 有运营会场要导流 → 运营型灵动岛(同频色,有背景动效,灵活)
  • 大促核心展示 → 大促型灵动岛(尺寸更大、异型背板,最抓眼球)
