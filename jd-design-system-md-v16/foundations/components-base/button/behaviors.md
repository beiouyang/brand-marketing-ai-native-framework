# 按钮 · 交互 / 禁止

> design.md → [主文档](./design.md) · 同 bundle: [spec](./spec.md) · [variants](./variants.md)

## 应用场景

### ✅ 用
- 表单提交 / 确认（下单、登录、支付、保存）→ 主要按钮
- 页面级首要 CTA（立即购买、立即领取、加入购物车）→ 主要按钮
- 弹窗主操作（确认删除、完成、知道了）→ 主要按钮
- 并列双操作（加购 + 立即购买）→ 次要(金) + 主要(红)

### ❌ 不用
- 同页面 / 同区域多个主按钮并排（应仅 1 个主按钮，其余降级）
- 非首要操作（取消 / 返回 / 关闭 → 次要 / 文字按钮）
- 列表行内操作（用图标按钮 / 文字按钮，避免抢戏）

## 交互

- **状态切换**：默认 → 按下（色加深）→ 抬起回默认；禁用态不可点、无反馈。
- **禁用语义**：禁用用灰（`#B4B8BF`）或保留色浅化两种——灰=完全不可用，浅化=暂不可用但保留样式语义。
- **深浅跟随系统**：浅 / 暗随系统外观；中性 / 金 / 浅红样式取暗色独立 token，品牌红浅暗一致。
- **图标 slot**：前 / 后置图标可选，随文字色 currentColor。

## Donts

- ❌ 不要硬编码 `#FF0F23` 替代 `color_primary` —— 主色由 Foundation 集中管理，暗色 / 换肤依赖 token 解析。
- ❌ 不要在非首要场景用主要按钮 —— 同页应仅 1 个主按钮，过多稀释视觉权重。
- ❌ 不要把按钮圆角改成与尺寸档不符的值 —— 圆角是尺寸分组的产物（48→8 / 40→6 / 更小→4），非单按钮可调。
- ❌ 不要把字号字重随意改 —— 各尺寸字号字重是组件契约（48=18/600 等）。
- ❌ 暗色不要直接复用浅色金 / 中性 / 浅红样式 —— 暗色独立取值（金 `#4A3B29`/`#B38B6D`）。
- ❌ 按下高亮 overlay `#FFFFFF@90%` 不要换其它透明度 —— 暂为 rgba-suggestion，待 `atom.white_90` 落定切回 token。

## 多端适配

随系统深浅切换；尺寸档适配不同密度场景（满宽 48 / 紧凑 40 / 行内更小）。
