1组件定义

一句话：页面底部承载该页主操作的悬浮栏，随业务页切换布局与信息密度，支持浅 / 暗双态。

页面底部悬浮的操作承载栏 —— 在商详 / 购物车 / 结算 / 订详 / 店铺等业务页承载该页的主操作（立即购买 / 去结算 / 立即支付 等），随页面场景切换布局与信息密度，支持浅色 / 暗色双态。

与 Tabbar / Button 的边界

组件职责关系
底部工具栏（本）业务页底部的操作 + 信息聚合栏（按钮 / 价格 / 结算 / 凑单进度）——
Tabbar全局 tab 导航（首页/分类/购物车/我的）+ Joy Agent + 灵动岛同属「底部导航栏」Relay 文件的不同画板；职责不重叠
Button单个按钮原子本组件内操作按钮复用 Button 原子
二者都贴底，但 Tabbar 是「去哪」（导航），Action Bar 是「做什么」（操作）。

2行为准则

核心约定：主操作唯一（红）；底部留 34px 安全区；浅 / 暗双态各取独立中性色；价格用京东正黑数字体强调。

  • 主操作优先 —— 每个场景最多一个主要操作（红），次要走金 / 灰；双操作时主操作恒定靠右。
  • 安全区 —— docked bar 底部留 34px（padding-bottom），适配全面屏 Home Indicator。
  • 深浅跟随系统 —— 浅 / 暗双态随系统外观切换，中性色按暗色独立 token 取值（非浅色反相）。
  • 凑单进度 —— 购物车凑单条底部进度条随实时优惠进度变化（实填宽度 = 进度百分比）。
  • 价格强调 —— 价格用京东正黑 V2.3 数字体 + 主操作红，是栏内视觉重心。

3设计属性

两类容器：悬浮态 docked bar（毛玻璃）/ 结算栏（实心白条）。4 类按钮色（红 / 金 / 灰 / 医保蓝）× 3 尺寸档（44 / 40 / 36）。暗色为独立态。

Donts

❌主操作不唯一

不要在一个操作栏放多个主要操作（红按钮）——主操作唯一。

❌暗色复用浅色中性色

全选 / 副说明 / 容器底 / 选择框填充在暗色下都换值，不能直接套浅色。

❌进度条画成虚线

凑单底部是实心进度填充（满减进度），不是撕边虚线 / dashed。

❌副说明整条同色

「查看明细 + 白条优惠」是中性灰 + 优惠红分段，不要整条涂红。

❌价格用 PingFang 顶替

价格数字须用京东正黑 V2.3，字形 / 等宽观感与 PingFang 不一致。

❌按钮被长副说明挤出栏外

长副说明时左区可收缩，去结算等右侧按钮 flex-shrink:0 不被挤出 351 边界。
