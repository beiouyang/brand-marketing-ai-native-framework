# 圆角 · 视觉与结构规格

> design.md → [主文档](./design.md) · 同 bundle: [variants](./variants.md) · [behaviors](./behaviors.md)

## 圆角 token 继承

营销场景默认继承 V16.0 圆角体系，不额外扩展圆角梯度。

| V16 atom | V16 token | 值 | 营销场景继承策略 |
|---|---|---:|---|
| `Radius_16` | `radius_xxl` | 16px | 弹层 / 弹窗 / 底导等悬浮层中大面积组件。 |
| `Radius_12` | `radius_xl` | 12px | 顶导 / 吐司提示等悬浮层小面积组件。 |
| `Radius_8` | `radius_l` | 8px | 导购型页面卡片结构 / 单行高度 40 及以上组件。 |
| `Radius_6` | `radius_base` | 6px | 单行高度 28 至 36 的组件。 |
| `Radius_4` | `radius_s` | 4px | 单行高度 20 至 24 的组件。 |
| `Radius_2` | `radius_xs` | 2px | 标签。 |
| `Radius_0` | `radius_xxs` | 0px | 平台型页面通栏卡片结构。 |

## 应用判断

| 判断项 | 规则 |
|---|---|
| 是否为系统级基础组件 | 是：回到 V16 组件语义与高度规则；否：继续判断结构角色。 |
| 是否形成独立结构盒 | 是：按页面框架、楼层、卡片、营销弹窗等结构层级选圆角；否：优先使用间距、分割线、背景差。 |
| 结构层级是否更高 | 层级越高、独立性越强，圆角越大。 |
| 信息是否附属 | 层级越低、信息附属性越强，圆角越小。 |
| 是否为平台型通栏结构 | 可使用直角，以保持页面秩序与信息效率。 |

## 结构角色

| 角色 | 包含 | 圆角策略 |
|---|---|---|
| 系统级基础组件 | 标签、按钮、气泡、系统级提醒、弹窗 | 按 V16 基础组件语义与单行高度规则映射。 |
| 页面框架与模块 | 页面整体框架容器、楼层、卡片、营销弹窗 | 按结构层级、独立性与内容边界选择圆角。 |
| 非独立结构盒 | 仅做内容分隔的组合区域 | 优先使用间距、分割线、背景差，不额外制造圆角。 |

## 章节原文

### 基础规范继承

基于 JD APP 16.0 设计系统-圆角进行营销场景的继承。圆角梯度应用于页面结构、容器层级与组件边界表达，不参与视觉图形风格定义；具体圆角规则与完整梯度定义详见《16.0 设计系统-基础设计规范》。

继承如下内容：

- 圆角梯度
- 应用映射
- 应用规则

营销场景默认遵循 16.0 圆角规范，不额外扩展圆角梯度。

### 应用原则

系统级基础组件具备固定的交互语义与形态表达，包含用于标签、按钮、气泡等基础元素，以及系统级提醒与弹窗。

页面框架与模块用于形成独立结构盒，建立页面层级与内容边界，包含页面整体框架容器，以及框架内部模块容器，如楼层、卡片、营销弹窗等。

当定义圆角时，应先判断其使用角色。
