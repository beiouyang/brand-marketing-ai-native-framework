# 圆角 · 形态与场景维度

> design.md → [主文档](./design.md) · 同 bundle: [spec](./spec.md) · [behaviors](./behaviors.md)

## 维度总览

| 维度 | 取值 | 说明 |
|---|---|---|
| 规范来源 | 继承 V16.0 圆角规范 | 营销场景不新增梯度，只复用 V16 基础圆角体系。 |
| 使用角色 | 系统级基础组件 / 页面框架与模块 | 先判断圆角作用对象，再映射到 V16 圆角规则。 |
| 结构层级 | 高层级 / 中层级 / 低层级 / 通栏结构 | 层级越高、独立性越强，圆角越大；通栏结构可直角。 |
| 图形风格 | 不参与 | 圆角不作为营销视觉图形风格扩展。 |

## 使用角色

### 系统级基础组件

具备固定交互语义与形态表达的元素应回到 V16 基础组件规则，不在营销系统内单独维护圆角。

常见对象：

- 标签
- 按钮
- 气泡
- 系统级提醒
- 弹窗

### 页面框架与模块

用于形成独立结构盒，建立页面层级与内容边界。

常见对象：

- 页面整体框架容器
- 楼层
- 卡片
- 营销弹窗

## 结构层级

| 层级 | 特征 | 圆角倾向 |
|---|---|---|
| 高层级 | 独立性强，承载页面级或悬浮层内容 | 使用更大的 V16 圆角值。 |
| 中层级 | 形成模块或卡片边界 | 使用中等圆角值，并保持同层级统一。 |
| 低层级 | 信息附属性强，常作为局部标签或小控件 | 使用更小圆角值。 |
| 平台型通栏 | 强调秩序、效率和连续阅读 | 可使用直角。 |

## 组合关系

| 组合 | 说明 |
|---|---|
| V16 基础规范 + 营销继承说明 | 当前规范不是新的 token 定义页，而是营销场景的继承与应用原则页。 |
| 系统级基础组件 + V16 映射 | 具有固定组件语义的对象按 V16 语义与高度规则映射。 |
| 页面框架与模块 + 层级判断 | 独立结构盒按层级和独立性选择圆角。 |
| 非独立结构盒 + 间距 / 分割线 / 背景差 | 不形成结构盒时，不通过圆角制造额外层级。 |

## 缺省状态

圆角不因默认、按下、禁用、加载等交互状态发生变化。交互状态可影响颜色、透明度、描边、阴影或反馈动效，但不应改变同一组件或同一结构层级的圆角表达。
