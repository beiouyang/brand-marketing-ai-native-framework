## 本次识别范围

本次识别的是 Marketing Design System 中的「文字 Typography」规范页。它不是单一交互组件，而是一页 Foundation 类型的营销文字规范，用于说明 JD APP 16.0 基础文字系统在营销场景中的继承、扩展和使用边界。

已识别范围：

- 基础规范继承：继承 JD APP 16.0 基础文字规范中的字体家族、基准字号梯度、文本间距规则、字重体系。
- 文字类型拓展：区分前端动态渲染文字与视觉切图 / 图形资源配置文字。
- 文字应用扩展：覆盖数字表达、系统字体字号扩展、营销图形化字体和多种风格字体案例。
- 应用原则：明确系统字体优先用于动态渲染与阅读型内容，图形化字体优先用于视觉强化与品牌表达场景。

不在本次范围：

- 完整基础 Typography token 梯度定义，需引用 JD APP 16.0 基础设计系统。
- 字体授权、字体包下载地址、CDN 资产链接。
- 单个业务页面或业务组件的结构规格。
- 暗色、多语言、无障碍、大字版等专项适配。

## 变体 / 状态维度

本节点不是状态型组件，主要维度是「实现方式」「内容类型」「字体类型」「应用层级」。

| 维度 | 取值 | 说明 |
|---|---|---|
| 实现方式 | 前端动态渲染 / 视觉切图配置 | 动态文本走系统字体与基础规范；图形化字体用于主视觉、广告、营销装饰文本。 |
| 内容类型 | 阅读型内容 / 常规信息 / 视觉强化文案 / 数字表达 | 根据可读性、氛围表达和业务转化强度选择不同文字策略。 |
| 字体类型 | 系统字体 / 数字表达 / 图形化字体 / 风格字体 | 系统字体保证阅读效率；图形化字体服务品牌氛围和营销视觉表达。 |
| 应用层级 | 主标题 / 价格数字 / 权益信息 / 倒计时 / 商品信息 / 规则说明 | 越接近转化核心，越需要控制层级、强度和信息密度。 |

| 组合 | 说明 |
|---|---|

| 系统字体 + 图形化字体 | 系统字体负责动态渲染和阅读效率；图形化字体负责主视觉和品牌氛围。 |
