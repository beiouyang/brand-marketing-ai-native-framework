# 文字 · 交互 / 禁止 / 适配

> design.md → [主文档](./design.md) · 同 bundle: [spec](./spec.md) · [variants](./variants.md)

## 应用场景

### ✅ 用

| 场景 | 做法 |
|---|---|
| 动态渲染文字 | 使用系统字体与基础阅读型规范。 |
| 商品信息 / feeds / 商卡 | 优先保证阅读效率和信息扫描速度。 |
| 优惠、价格、补贴、倒计时 | 使用数字表达扩展突出核心利益点。 |
| 主视觉 / 广告位 / 主题会场 | 可使用图形化字体强化主题、氛围和品牌表达。 |
| 风格字体案例展示 | 使用 PNG / SVG 切图作为规范页主展示，避免因字体缺失回退为系统字体。 |
| 固定主题文案 | 可使用图形化字体或切图表达品牌、品类、节庆、科技等情绪。 |
| 多行说明文本 | 使用合理行间距控制阅读节奏。 |

### ❌ 不用

| 场景 | 原因 |
|---|---|
| 动态文本使用图形化字体切图 | 内容不可变更、不可搜索、不可适配，影响维护和可访问性。 |
| 阅读型正文使用复杂装饰字体 | 降低阅读效率，影响信息理解。 |
| 多个核心信息同时高强度强化 | 页面焦点混乱，削弱转化信息。 |
| 仅为装饰而放大字体 | 占用视觉资源，干扰业务信息层级。 |
| 规范页只用 HTML 字体名模拟风格字体 | 访问者本机缺少字体时会回退系统字体，无法稳定还原设计稿。 |

## 交互

本规范本身不定义点击、跳转、手势等交互。涉及倒计时、价格刷新、优惠变化等动态内容时，应保证：

1. 数字变化不造成布局抖动。
2. 单位、货币符号、补充说明与主数字层级清晰。
3. 动态内容必须由前端渲染，不使用静态切图承载。
4. 异常、缺省、加载态应回退到基础系统字体。
5. 固定风格字体案例在规范页中以切图展示；字体名、使用场景、Relay 节点以文本补充，便于检索与追溯。

## Donts

| Dont | 说明 |
|---|---|
| 不把图形化字体用于动态文本 | 图形化字体适用于视觉强化，不适合可变内容。 |
| 不把数字扩展当作纯视觉强化字号 | 数字扩展仍属于基础阅读型体系，应服务权益表达。 |
| 不在阅读型模块过度使用高强度字重和颜色 | 商卡、feeds、商品信息等场景应优先保证读完和扫读效率。 |
| 不让字体风格先于信息目标 | 字体风格应服务品类、主题和转化信息，而不是独立装饰。 |
| 不用未登记来源的字体切图作为正式资产 | 案例切图需要在 `_assets-cdn.md` 登记来源、尺寸、CDN 与授权状态。 |

## 多端适配

| 维度 | 规则 |
|---|---|
| 屏幕宽度 | 动态文本随容器自适应，避免固定切图导致溢出。 |
| 字号缩放 | 阅读型内容应跟随基础设计系统的字号与行高策略。 |
| 多语言 | 动态文本优先系统字体；图形化字体需另行确认多语言资产。 |
| 无障碍 | 重要信息不得只依赖图片文字；核心文案应保留可读文本。 |
| 降级 | 字体资产缺失时回退系统字体，并保持信息层级。 |
| 发布页资产 | 风格字体切图需提供 alt 文案，并同步保留字体名称、场景说明和示例文案。 |

## 章节原文（结构化摘录）

- 系统字体优先用于动态渲染与阅读型内容。
- 图形化字体优先用于视觉强化与品牌表达场景。
- 字体强化应优先服务于优惠力度、价格、主标题等核心信息表达。
- 多行文本场景下，应通过合理行间距控制阅读节奏。
- 避免多个高强度信息同时强化，影响页面视觉节奏与浏览体验。
- 风格字体案例在发布页中使用静态切图作为主展示，HTML 字体预览只能作为辅助。
