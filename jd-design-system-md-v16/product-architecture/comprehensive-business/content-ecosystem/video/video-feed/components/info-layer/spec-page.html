1组件定义

信息区 Info Layer 是沉浸式视频页面左下角的轻量文案承载层，用于展示发布者昵称、视频文案、原创标签和达人标签，并通过「展开 / 收起」承接被省略的长文案。

一句话理解：视频画面左下角那一小块白字——发布者是谁、这条视频在讲什么。文案太长就一行省略 + 「展开」，它只负责说清内容，不放点赞评论这类按钮。

信息区只承载内容说明和轻交互入口，不承载点赞、评论、分享、音量等强操作。右侧互动区、底部评论栏、播放进度线都应与信息区保持独立层级，避免点击区域重叠。

模块是否属于信息区说明
昵称✅ 是发布者或账号名称，固定在信息区首行
视频文案✅ 是默认单行截断，支持展开 / 收起
展开 / 收起入口✅ 是文案超过默认展示行数时出现
原创标签◻️ 可选与文案同排，位于文案前置位置
达人标签◻️ 可选与昵称同排，位于昵称右侧；当前样例为「数码达人」
互动按钮❌ 否归属 interactive-area
底部评论输入 / 通条❌ 否归属 BottomTab

核心边界：信息区固定锚定视频左下区域，右侧不得侵入互动区，展开文案不得覆盖底部评论栏与进度线。

2行为准则

核心约定：① 默认收起、文案只显示 1 行；② 文案没超出就不显示「展开」；③ 点「展开」看双行全文、入口变「收起」；④ 切下一条视频，文案一律回到收起态，不继承上一条的展开状态。

  1. 默认收起：页面初始进入时文案默认收起，展示 1 行，超出部分省略。
  2. 按需出现入口：文案未超出 1 行时不展示「展开」；超出 1 行时在文案行右侧展示「展开」。
  3. 点击展开：点「展开」后文案从 1 行扩展到双行形态，入口文案从「展开」切换为「收起」；展开态保留原创标签和达人标签，不新增商品卡等复杂模块。
  4. 点击收起：点「收起」后文案恢复 1 行省略，入口恢复为「展开」。
  5. 切换视频复位：上滑 / 下滑切换视频时信息区整体淡出并按方向位移；新视频渲染后文案必须复位为收起态，展开状态不跨视频继承。
  6. 展开不越界：展开态容器高度随文案增长，但不得覆盖底部栏主要点击区、进度条或右侧互动区。
  7. 标签仅展示：原创标签、达人标签默认仅展示；如业务接入可分别跳转原创内容说明 / 达人认证说明。
  8. 命中区域：展开 / 收起最小命中 44×28 DP，可通过透明 padding 扩大；昵称建议命中高度不小于 24 DP。

动效参数

场景动效参数
上滑切换视频向上 22 DP 并淡出220ms cubic-bezier(0.2, 0, 0, 1)
下滑切换视频向下 22 DP 并淡出220ms cubic-bezier(0.2, 0, 0, 1)

可访问性：展开入口用可聚焦按钮语义，aria-expanded 反映当前状态；无障碍文案应包含动作和对象，例如「展开视频文案」「收起视频文案」。

3组件类型

信息区由 4 个组合维度叠加而成：是否省略、单行 / 双行、是否带原创标签、是否带达人标签。Relay 共定义 9 个 symbol 覆盖典型组合。

维度取值说明
省略有 / 无有省略时文案末尾显示 …，并出现「展开」入口
行数单行 / 双行单行 symbol 高 46 DP，双行 symbol 高 64 DP
原创标签有 / 无当前样例为「原创」，位于文案前置位置
达人标签有 / 无当前样例为「数码达人」，位于昵称右侧

3.2 组合优先级

当多个内容同时存在时，按以下顺序保留：

  1. 昵称必须展示。
  2. 达人标签仅跟随昵称展示，不挤压昵称主体。
  3. 原创标签优先展示在文案前置位置。
  4. 长文案优先使用「省略=有，行数=单行」形态，通过「展开」承接。
  5. 展开后按「省略=无，行数=双行」形态展示，不跨视频保留。

4组件结构

信息区为 纵向两行结构：昵称行在上、文案行在下，行间距 8 DP；不使用独立背景色 / 圆角 / 描边，文字依赖阴影保证可读性。

基础结构：上面一行是昵称（可带达人标签），下面一行是文案（可带原创标签 + 展开入口）。整块透明叠在视频上，靠白字 + 投影看清。具体 DP / Token 见专业版。

信息区适用于沉浸式视频流、短视频、直播切片等需要在画面上叠加内容说明的场景。下表覆盖典型组合与状态。

多端适配

场景规则
375 DP 宽机型使用 274 DP 基础宽度，右侧避开互动区

7错误示例

DON'T 把互动按钮放进信息区

问题：将点赞、评论、分享、关注等强操作合并到信息区。这些归属 interactive-area / BottomTab，混入会让点击区域与右侧互动区、底部栏重叠。

DON'T 展开文案遮挡底部栏 / 进度条

问题：展开后容器高度无限增长，覆盖底部评论输入、内容通条、进度条或系统安全区。展开态必须在不越界的前提下增高。

DON'T 给信息区整体加不透明底板

问题：为增强可读性给整个信息区套整块蒙层。应优先使用文字阴影；标签只用 rgba(255,255,255,0.10) 局部半透明底。

DON'T 短文案时常驻「展开」

问题：文案未超出 1 行也展示「展开」入口。入口只在文案确实溢出时出现。

DON'T 切换视频后保留上一条的展开状态

问题：上滑切到新视频后，文案仍停留在已展开形态。展开状态不跨视频继承，新视频必须复位为收起态。

DON'T 新增 Relay 未定义的复杂模块

问题：在信息区内塞入商品卡、切片定位等复杂运营模块。当前节点只定义文案、原创标签和达人标签。
