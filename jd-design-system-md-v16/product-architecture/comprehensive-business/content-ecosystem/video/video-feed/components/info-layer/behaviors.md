# 信息区 · 行为 / 禁止 / 适配

> design.md → [index](./design.md) · 同 bundle: [spec](./spec.md) · [variants](./variants.md)

## 应用场景

### ✅ 用

- 沉浸式视频流、短视频、直播切片等需要在画面上叠加内容说明的场景。
- 需要展示账号昵称、视频文案、原创标签或达人标签的轻信息场景。
- 视频切换时需要信息层跟随内容整体刷新、淡出淡入的场景。

### ❌ 不用

- 需要承载强操作按钮的场景，例如点赞、评论、分享、关注按钮组。
- 需要展示完整长正文、图文详情、商品卡或复杂表单的场景。
- 需要持续占据底部大面积空间的场景，应改用底部通条、半屏弹层或详情页。

## 文案展开 / 收起

### 默认收起

- 页面初始进入时，文案默认收起。
- 收起态文案展示 1 行，超出使用省略。
- 当文案未超出 1 行时，不展示「展开」入口。
- 当文案超出 1 行时，在文案行右侧展示「展开」。
- 若配置了原创标签，标签位于文案行前置位置；若配置了达人标签，标签位于昵称右侧。

### 点击展开

- 点击「展开」后，文案从 1 行扩展到 Relay 定义的双行形态。
- 入口文案从「展开」切换为「收起」。
- 展开态容器高度随文案增长，但不得覆盖底部栏主要点击区。
- Relay 原稿只提供「展开」入口和双行展示样例，未提供「收起」节点；工程需使用同字号、同色值补齐「收起」入口。
- 展开态保留原创标签和达人标签，不新增商品卡等复杂模块。

### 点击收起

- 点击「收起」后，文案恢复 1 行省略。
- 入口文案恢复为「展开」。
- 收起后恢复 Relay 对应的 `省略=有，行数=单行` 变体。

### 切换视频复位

- 上滑 / 下滑切换视频时，信息区整体跟随内容淡出并按方向位移。
- 新视频渲染后，文案必须复位为收起态。
- 展开状态不跨视频继承。
- 新文案需要重新判断是否展示「展开」入口。

## 点击区域

| 元素 | 点击行为 | 命中区域 |
|---|---|---|
| 昵称 | 可跳转账号主页，未接入时仅展示 | 建议高度不小于 24 DP |
| 文案正文 | 默认不跳转；展开态可跟随文本区域收起由业务决定 | 不应抢占右侧互动区 |
| 展开 / 收起 | 切换文案状态 | 最小命中 44×28 DP，可通过透明 padding 扩大 |
| 原创标签 | 默认仅展示；如业务接入，可跳转原创内容说明 | 标签整体可点 |
| 达人标签 | 默认仅展示；如业务接入，可跳转达人认证说明 | 标签整体可点 |

## 动效

| 场景 | 动效 | 参数 |
|---|---|---|
| 上滑切换视频 | 信息区向上 22 DP 并淡出 | 220ms `cubic-bezier(0.2, 0, 0, 1)` |
| 下滑切换视频 | 信息区向下 22 DP 并淡出 | 220ms `cubic-bezier(0.2, 0, 0, 1)` |
| 展开文案 | 高度从 46 DP 展开到 64 DP | 180ms ease |
| 收起文案 | 高度回收，文本单行省略 | 160ms ease |

动效不应改变左下锚点和容器宽度。切换视频时，信息区与右侧互动区使用同方向位移，保持页面整体感。

## 内容更新

- `nickname` 随 feed 切换立即替换。
- `description` 随 feed 切换立即替换，并重新进入收起态。
- 若新 feed 无文案，隐藏文案行与展开入口。
- 若新 feed 有原创标签或达人标签，按 Relay 对应变体渲染。
- 异步数据晚到时，不要让信息区高度突然覆盖底部栏；应在收起态内完成更新。

## 可访问性

- 信息区容器使用 `aria-label="info-layer"` 或业务语义标签。
- 展开入口应使用可聚焦按钮语义，`aria-expanded` 反映当前状态。
- 展开入口的无障碍文案应包含动作和对象，例如「展开视频文案」「收起视频文案」。
- 点击区域小于 44 DP 时，应使用透明 padding 扩大命中范围。

## Donts

- ❌ 不要让「展开」入口在短文案时常驻展示。
- ❌ 不要在切换视频后保留上一条视频的展开状态。
- ❌ 不要把分享、评论等互动按钮合并到信息区。
- ❌ 不要让展开文案遮挡底部评论输入、内容通条或系统安全区。
- ❌ 不要在明亮视频上取消文字阴影。
- ❌ 不要在本组件内新增 Relay 未定义的商品卡、切片定位或复杂运营模块。

## 多端适配

| 场景 | 规则 |
|---|---|
| 375 DP 宽机型 | 使用 274 DP 基础宽度，右侧避开互动区 |
| 更窄屏 | 优先缩短文案可用宽度，保留展开入口 |
| 更宽屏 | 左边距可随页面安全区增加，但信息区宽度不无限扩展 |
| 大字版 | 昵称 / 文案按系统字号放大时，默认收起态仍保持 1 行，展开态优先保持 2 行 |
| 深浅视频背景 | 统一保留白字 + 阴影策略，不根据视频动态换色 |

## 工程验收

- 默认态 DOM 至少包含昵称、文案和按需出现的展开入口。
- 长文案点击「展开」后能展示更多内容，再次点击「收起」能恢复单行省略。
- 切换视频后，已展开的文案必须回到收起态。
- 原创标签、达人标签应能分别组合到 Relay 定义的对应变体。
- 右侧互动区和底部栏的点击区域不得被信息区覆盖。
