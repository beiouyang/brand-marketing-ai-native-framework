1组件定义

InteractiveArea 是短视频沉浸流右侧互动栏的完整组合组件，把作者头像入口、点赞、评论、可选种草、转发与音量控制组织成一条稳定的垂直 Action Rail。

它的核心边界是组合顺序、变体开关与事件透传，而不是重新定义头像、图标、文字、颜色、阴影或音量按钮的视觉细节。

对象职责视觉来源
interactive-area定义 rail 顺序、with/without grass 变体、事件 payload 与页面边界。不新建视觉；复用子组件契约。
InteractiveAvatar维护头像、直播标签、直播环、关注关系 2×3 状态矩阵。references/InteractiveArea/InteractiveAvatar.md
InteractiveActionItem / Volume维护互动图标、计数文字、点赞/种草激活态与音量控制。references/InteractiveArea/InteractiveActionItem.md + foundations/visual/Icon.md

立场：interactive-area 只改写逻辑样式和发布结构，不改变原互动区视觉规范；旧目录里的 HTML 产物不作为新规范源。

2行为准则

  1. 顺序固定：始终按 avatar → like → comment → optional grass → share → volume 渲染。
  2. 变体不占位：without_grass 不渲染种草空容器；with_grass 只在评论与转发之间插入种草。
  3. 头像单源：liveState 与 relationship 透传给 InteractiveAvatar，组合层不改头像状态层。
  4. 操作项单源：like/comment/grass/share 透传 type、isActive、disabled、count。
  5. 音量独立：volume 不复用普通 ActionItem 高度、计数或激活色，只使用音量控制二级组件。
  6. 页面拥有定位：右侧位置、安全区、z-index 与播放器状态由页面层处理。
  7. 禁用无副作用：disabled 操作不触发业务事件，也不改变视觉状态。
  8. 图标用真资产：互动区图标必须引用 assets/icons/*.svg，并遵循 Icon.md 的 50×50 interactive 画板规则。

3组件类型

interactive-area 定义 2 个组合变体，并复用头像、操作项和音量控制的状态维度。视觉变化来自子组件，组合层只决定渲染哪些 slot。

4组件结构

InteractiveArea 由 1 个头像子组件、3 或 4 个互动操作项，以及 1 个音量控制构成。互动 icon 使用 Icon.md 的 size="interactive" 规则：SVG 画板固定 50×50，内部核心图形约 27pt，透明热区不可裁剪。

正例

GOOD 种草能力开启时只插入一个 grass slot

使用 with_grass，grass 位于 comment 与 share 之间；其他子组件视觉保持原样。

GOOD 无种草时总高度自然收缩

without_grass 不保留空节点，不用透明占位维持旧高度。

GOOD 音量用独立二级组件

muted 使用 50×50 hotzone；unmute_prompt 使用 69×30 胶囊，不复用 size.action_item。

7错误示例

BAD 在组合层重绘 icon

问题：会绕过 Icon.md 的 50×50 interactive 画板、透明热区和 currentColor 收口规则。

BAD without_grass 保留透明种草位

问题：破坏 Relay 组合节奏和总高度公式，也让业务状态与视觉结构不一致。

BAD 用普通 24px 图标替代互动区 SVG

问题：互动区图标内部核心约 27pt，缩到常规尺寸会显得过小，并丢失 50×50 热区。

API 速查

API / Props 类型定义为专业版内容，常规版暂不展示。

type InteractiveAreaVariant = "without_grass" | "with_grass";
type AvatarLiveState = "not_live" | "live";
type AvatarRelationship = "unfollowed" | "intermediate" | "followed";
type VolumeState = "muted" | "unmute_prompt";

interface InteractiveAreaProps {
  variant: InteractiveAreaVariant;
  avatar: { liveState: AvatarLiveState; relationship: AvatarRelationship };
  like: ActionItemProps;
  comment: ActionItemProps;
  grass?: ActionItemProps;
  share: ActionItemProps;
  volume: { state: VolumeState };
  onActionTap?: (payload: { type: "like" | "comment" | "grass" | "share"; count?: number }) => void;
  onVolumeToggle?: (payload: { previous_state: VolumeState; next_state: VolumeState }) => void;
}
