1组件定义

BottomBar 是视频详情页底部吸底组件族，用于承载轻量评论入口与视频内容相关的复合信息通条。它不是京东首页 Tabbar，也不是一级导航 Tabbar；组件边界限定在视频详情页底部的评论输入区、信息导流条和复合运营入口。

一句话理解：视频页最底部那一条——要么是评论入口，要么是跟当前视频相关的信息条（话题/合集/短剧/热榜等）。两者属于同一个组件族，不是两个独立模块。

对比维度BottomBar（本组件）首页 Tabbar
定位视频详情页内的评论入口 + 内容导流条APP 一级导航 Tab 切换
使用场景视频消费场景全局常驻
形态单条通条 / 评论输入态多坑位 icon + 文字

核心边界：BottomBar 仅服务于视频详情页底部，不可作为全局导航、页面级 Tab 或购物车入口使用。

2行为准则

  1. 同族切换：评论入口与复合通条属于同一 BottomBar 组件族，由业务场景决定展示哪种形态。
  2. 单条优先：生产页面同一时刻优先展示一条通条；多条入口需经过优先级仲裁。
  3. 内容相关：通条只承载与当前视频强相关的轻量入口，不覆盖视频主体和右侧互动区。
  4. 结构稳定：icon → 类型名 → 分隔符 → 内容 → 辅助信息 → 箭头，顺序不可被业务重排。
  5. Token 驱动：背景、文字、间距、圆角、字体必须映射至已有 Token；缺失项记录为待补。
  6. 私有资产收敛：非全局通用图标与运营标签保留在 assets/，不沉淀为全局 Icon。

BottomBar 共定义 2 种主形态：Type A 评论输入态 + Type B 复合通条态。

4组件结构

BottomBar 由 Safe Area + Background Layer + Content Layer（Type A 或 Type B） 三层构成。

基础结构（常规版概述）：最外层是暗色吸底背景 + 安全区；内容层根据业务场景展示评论入口（Type A）或信息通条（Type B）。具体 Token / DP 见专业版。

正例

DO 单条通条，结构一致

话题/合集/短剧等不同业务类型统一使用相同通条结构：leading icon + label + sep + content + trail，保持视觉稳定性。

DO 私有图标收敛在 assets/

非全局通用的图标（话题#、三单票券、国补标签）保留在组件私有 assets/ 目录，不污染全局 Icon 库。

DO 单条优先，优先级仲裁

同一时刻只展示一条通条。当多个业务都想展示时，经过优先级仲裁选出最相关的一条。

7错误示例

DON'T 通条内放无关入口

问题：在 BottomBar 通条中塞入购物车入口、商品推荐或与当前视频无关的运营弹窗。通条只服务当前视频内容。

DON'T 业务自行重排结构

问题：不同业务将 icon、label、content 的顺序或位置自行调整。结构顺序是固定契约，不可被业务方重排。

DON'T 多条通条同时展示

问题：多条通条在视频页底部同时堆叠展示（设计稿中多条并列仅为变体展示，非生产态）。
