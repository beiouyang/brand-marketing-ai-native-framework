# BottomBar · 行为规范

## 1. 基础交互

| 形态 | 默认行为 | 反馈 |
|:---|:---|:---|
| 基础评论输入态 | 点击评论入口打开评论面板或进入评论输入流程。 | 页面层展示键盘、评论面板或登录拦截。 |
| 表情 icon | 点击进入表情能力；若评论面板未打开，可先打开评论面板并聚焦表情。 | 能力不可用时隐藏，不保留空热区。 |
| 复合通条 | 点击整条进入对应内容或展开内容组。 | 使用页面跳转、半屏浮层或展开列表，由业务目标决定。 |
| 右侧箭头 | 与整条主行为一致，或在展开态执行收起。 | 箭头只表达方向和状态，不单独承载额外语义。 |
| 右侧 `下一集` 按钮 | 独立执行下一集。 | 不触发左侧信息条跳转。 |

## 2. 点击区域

| 区域 | 热区规则 |
|:---|:---|
| Type A 评论入口 | 整个输入框区域可点击；`icon-comment-check-brand.svg` 和占位文案属于同一热区。 |
| Type A 表情 icon | 调用 `icon-happy.svg`，可作为独立热区；热区不得小于视觉 icon 所在容器。 |
| Type B 单条整行 | 整条通条为一个主热区，leading、文案、辅助信息、箭头均指向同一目标。 |
| Type B 左右分栏 | 左侧信息条与右侧按钮为两个独立热区，点击事件不得互相冒泡触发。 |
| 展开态通条 | 如业务定义为收起，整条或箭头均可收起；若只允许箭头收起，需在业务层明确热区。 |

## 3. 跳转规则

| 内容类型 | 默认跳转 |
|:---|:---|
| 话题 | 打开话题聚合页或话题内容流。 |
| 相关搜索 | 发起搜索或打开搜索结果页。 |
| 合集 | 打开合集详情或合集播放列表。 |
| 短剧 | 打开短剧详情、短剧内页或剧集列表。 |
| 下一集 | 直接切换到下一集，不进入合集详情。 |
| 热榜 | 打开榜单聚合或对应热榜内容页。 |
| 会场优惠 | 打开会场承接页。 |
| 国补优惠 | 打开国补活动承接页。 |

## 4. 展开 / 收起规则

| 状态 | 规则 |
|:---|:---|
| 收起态 | 使用 arrow-right；点击后可跳转或展开，业务只能选择一种主行为。 |
| 展开态 | 使用 arrow-up；表示当前内容组已展开。 |
| 展开内容 | Relay 当前节点未给出展开面板结构；展开列表高度、动效、遮罩和列表项规格需待设计确认。 |
| 收起触发 | 可由再次点击通条、点击 arrow-up、切换视频、滚动或页面失焦触发。具体触发组合由业务场景确认。 |

## 5. 优先级

同一视频详情页底部同一时刻只建议展示一个 BottomBar 主内容。多个候选入口同时存在时，按以下优先级仲裁：

| 优先级 | 内容 |
|:---:|:---|
| P0 | 强业务控制：评论输入禁用、登录拦截、安全治理提示。 |
| P1 | 播放连续性：下一集、短剧内页、合集续播。 |
| P2 | 内容强相关：合集、短剧、话题、相关搜索。 |
| P3 | 强运营：热榜、会场优惠、国补。 |
| P4 | 默认评论输入态。 |

> 页面可根据业务目标调整 P2 / P3 顺序，但不得让运营入口长期挤占 P1 播放连续性入口。

## 6. 降级规则

| 异常 | 降级 |
|:---|:---|
| 复合通条数据缺失 | 回退到 Type A 基础评论输入态。 |
| leading icon 加载失败 | 使用同类默认私有 icon；仍失败则隐藏 leading 并保留文案。 |
| 运营标签 SVG 缺失 | 回退为普通文本标签，或隐藏该运营入口。不得使用未经确认的新色值重绘。 |
| 辅助信息缺失 | 隐藏辅助信息，不保留空占位。 |
| 右侧按钮不可用 | 回退为无按钮通条，主行为指向详情或聚合页。 |
| 箭头资源缺失 | 使用全局 arrow 资产；全局也缺失时隐藏箭头，但保留整条点击。 |
| 评论能力不可用 | 保留入口但点击触发页面级提示，或按业务策略禁用；不得在 BottomBar 内加入长说明。 |

## 7. 图标与资产行为

| 规则 | 说明 |
|:---|:---|
| 私有资产优先本地管理 | 复合组件前面的 icon 如非全局资产库已有，应放在当前组件 `assets/`。 |
| 通用箭头优先全局引用 | `arrow-right`、`arrow-up` 如全局已有，不在本组件重复切图。 |
| 标签资产直接使用 SVG | `8折`、`免费观看`、`国家补贴` 等文字标签如已有切图，直接引用资产，不拆为普通文本。 |
| 临时运营素材不进入全局库 | 仅在多个组件长期复用并经过设计系统确认后，才可迁移到全局 Icon。 |

## 8. 禁止用法

| 禁止项 | 原因 |
|:---|:---|
| 将 BottomBar 写成首页 Tabbar 或一级底部导航 | 本组件只服务视频详情页底部吸底评论 / 信息通条。 |
| 在一个 BottomBar 中同时堆叠多个复合通条 | Relay 多条为变体展示矩阵，不等于生产态可堆叠。 |
| 为不同业务任意重排结构顺序 | 会破坏 icon、类型、内容、辅助信息、箭头的稳定识别。 |
| 使用裸 CSS 色值覆盖 Token | 违反 Token 驱动原则，也会导致深色沉浸场景不可维护。 |
| 把运营临时素材沉淀为全局 Icon | 会污染全局资产语义。 |
| 右侧 `下一集` 与左侧信息条共用同一点击事件 | 会造成误触和跳转歧义。 |
| 在通条内放置长段落说明 | BottomBar 是轻量入口，不承担页面说明文案。 |
