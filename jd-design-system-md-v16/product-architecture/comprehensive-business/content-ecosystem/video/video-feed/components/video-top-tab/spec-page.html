1组件定义

VideoTopTab 是视频场景顶部导航与频道切换的复合组件，负责承载频道 Tab、选中游标、返回/搜索/菜单/二级跳转等导航槽位。

它的核心边界是「视频顶部频道导航 + 页面联动」，而不是通用 Tab，也不是全局图标系统。
对比维度VideoTopTab近邻组件
层级Component Composition LayerToken / Visual Atom

2行为准则

  1. 只组合既有能力：组件引用现有 token / visual atoms / private assets，不创建基础视觉属性。
  2. 点击切换单源同步：TabItem 点击后同步 selected state、indicator 与页面频道。
  3. 横向滚动不压缩文案：频道超出可视区时进入 scrollable，而不是缩小字体。
  4. TabList 右锚定：非沉浸流（常规 / 暗黑）从右往左露出视频 / 短剧 / 小说 / 推荐 / 直播；沉浸流 track 左移 68px，左侧露出「关注」边缘；均不压缩文案。
  5. 选中不染色：selected 文案与下方 indicator 继承当前模式文本色，不额外使用品牌色高亮。
  6. 吸顶不重排：sticky 只改变容器背景与层级，不改变 Tab 顺序和选中态。
  7. 沉浸态优先可读：视频背景上使用沉浸态文本与防穿透 shadow token。
  8. 活动态可降级：运营素材、Lottie 或活动 indicator 失败时回退为 selected + underline。
  9. Dropdown 与 TabList 单选一致：不允许两个区域出现不同 selected。
  10. 降噪保护核心频道：视频、直播、关注为常驻 Tab，不被运营配置挤出首屏。

4组件结构

VideoTopTab 由 Container + Navigation Slot + TabList + Indicator + Dropdown 组合而成。

基础结构： 左右是功能入口，中间是频道，底部用 indicator 告诉用户当前在哪个频道；活动、吸顶、沉浸都只是状态层，不改变这个骨架。

5组件设计属性

区域布局规则

Dropdown折叠频道层与 TabList 保持单选同步。

7错误示例

反例 用缩小字号解决频道过多

问题： 频道过多应进入 scrollable / folded / dropdown，而不是破坏 typography token。

反例 把私有 icon 迁移到全局 Icon

问题： 当前 4 个导航 icon 是 VideoTopTab component-owned private assets，不属于 global Icon system。
