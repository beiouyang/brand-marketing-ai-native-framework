1定义

Video Feed Page 是视频沉浸式信息流页面，用于承载短视频播放、频道切换、视频信息展示、右侧互动操作与底部评论输入。

它的核心边界是组合既有横向专项组件形成完整视频消费场景，而不是在页面层新增视觉组件、重写组件 token 或定义视频内容层的 Design System 组件。

一句话理解：这是一张全屏视频画布，页面只负责把顶部频道、左下信息、右侧互动和底部评论入口稳定地放在视频之上。
对象 主要内容 交互模型 布局模型
普通 Feed 页面 多条内容卡片列表 卡片级浏览、点击进入详情 文档流 / 列表流
Video Feed Page 单条全屏视频内容 上下滑切视频、叠加互动、沉浸播放 immersive overlay video feed layout

页面立场：Page 层保持 Page → Components → Visual Atoms → Tokens 的依赖链路，只声明组合、位置、状态联动和优先级。

2行为准则

  1. 内容优先：视频内容始终是 base layer，页面组件只作为 overlay 叠加。
  2. 固定锚点：video-top-tab、interactive-area、bottom-tab 在当前视频帧中保持固定锚点。
  3. 组件自治：Page 调用组件公开能力与状态参数，不进入组件内部子树修改 slot 顺序。
  4. 滑动同步：上下滑切视频时，页面负责同步当前视频的左下信息、右侧互动和底部评论上下文。
  5. 键盘避让：进入 keyboard-visible 后，底部输入区避让键盘，信息层避免覆盖输入主路径。
  6. Sheet 协调：评论、分享、更多频道等 BottomSheet 打开时，页面协调底部入口、信息层与手势区域。
  7. 手势优先级：上下滑优先作用于视频 feed；右侧点击操作不得触发视频切换。
  8. 禁止 raw value：页面不直接定义位置、间距、颜色、圆角、阴影或动效的 raw value。

3类型

本页是单一页面模式，按场景状态区分为 7 个页面状态：idle、playing、paused、commenting、immersive、keyboard-visible、full-screen。

状态 影响组件 规则
idle all 初始页面结构已挂载，视频内容未进入强播放反馈。
playing info-layer / interactive-area / bottom-tab 保持沉浸 overlay 展示，不阻断视频内容。
paused interactive-area / info-layer 互动区仍可点击，页面可保留信息层可读性。
commenting bottom-tab / info-layer / BottomSheet 评论入口触发输入或评论面板，信息层根据空间避让。
immersive video-top-tab / interactive-area / info-layer 顶部、右侧、左下内容以沉浸态视觉叠加在视频上。
keyboard-visible bottom-tab / info-layer / BottomSheet 底部输入区键盘避让，信息层避免与键盘和输入区冲突。
full-screen video-top-tab / bottom-tab / interactive-area 可收敛非必要 overlay，但不得修改组件内部结构。

不允许扩展：页面不新增替代型导航、互动区、评论入口、Icon、Avatar 或视频内容组件。

4结构

VideoFeedPage 由视频内容层、顶部频道、左下信息、右侧互动、底部评论入口和系统安全区组成。

VideoFeedPage
├── video-top-tab
├── info-layer
├── interactive-area
├── bottom-tab
└── SystemSafeArea
节点 页面职责
video-top-tab 页面顶部频道切换、频道导航、搜索 / 菜单入口，在沉浸视频上方 overlay 展示。
info-layer 左下信息展示，包括昵称、视频标题、商品信息、视频切片等内容。
interactive-area 右侧点赞、评论、种草、分享、头像关注、音量控制等互动操作。
bottom-tab 底部评论输入入口、表情入口与评论触发区域。
SystemSafeArea 页面级系统安全区处理，包括顶部状态栏、底部 Home Indicator、键盘避让。
静态预览引用同目录 index.html，用于展示页面组合关系。

6正反案例

GOOD 沉浸视频消费页

视频内容保持为主层，顶部频道、左下信息、右侧互动、底部评论入口以 overlay 方式稳定可达。

  • 组件职责清晰，Page 不改写组件内部结构。
  • 互动入口在播放、暂停、评论等状态下仍能保持操作优先级。

GOOD 评论与键盘联动

点击 bottom-tab 后进入 commenting 或 keyboard-visible，页面负责避让输入区并协调 BottomSheet。

  • 底部入口拥有最高交互优先级。
  • info-layer 根据空间让出输入路径。

BAD 在页面层新增组件或 atom

问题：绕过 existing component system 自绘 video-top-tab、info-layer、interactive-area 或 bottom-tab，会破坏组件职责边界与 token 追溯。

BAD 用 raw value 写页面级布局

问题：直接在 Page 文档中定义位置、间距、颜色、圆角、阴影或动效的 raw value，会绕开 V16 token 体系。

BAD 将视频内容包装成 Design System 组件

问题：Video Content 是业务内容层，不作为 Design System 组件新增；页面只声明其作为 base layer 的关系。
