1设计原则

👉 一句话理解:视频场域底导的单个坑位(那个 44×44 的小方块) —— 用户在 App 底部看到的「逛逛」「文案」按钮就是它。本组件只管这个小方块长什么样、有几种状态。整条底导由通用底导组件统一规范。

它解决什么:让视频场域(逛 / 视频频道)的底导坑位跟其他频道有区分 —— 选中态的图标是红色 video 播放图标(▶),其他频道是各自的品牌图标。这样用户一眼知道自己在哪个频道。

为什么独立录入:视频频道的选中视觉(红色 video icon + 红字加粗) 跟通用底导(线性 icon + 灰底高亮)不一样,需要单独说清楚。其他规则(2-5 坑位、文案 4 字、间距等)沿用通用底导,本文不重复。

视频场域底导单坑原子是 44×44 的单个坑位组件,作为通用 tabbar的 slot 内容,在视频/逛频道场景下使用。

它的核心边界是「单坑视觉(状态切换 + 招手位)」而非「整条底导布局」。整条底导的容器、形态、坑位数、安全区等规范继承通用 tabbar,本组件只定义视频场域单坑的视觉差异。

关键规则:选中态用红白实心 video icon + 红字 Semibold；icon / 营销态改为 38×38 圆形位图；招手位只允许红点、数字、文字三类 badge。

1.1 与通用 tabbar 单坑的差异

维度通用 tabbar 单坑视频场域单坑（本组件）
默认 icon各频道线性 icon（home / category 等）线性 video icon（频道占位为通用 home icon）
选中态视觉线性 icon + 灰底高亮红实色 video icon + 红字 Semibold（icon 形状由 line → filled，红方块包白 ▶）
容器底色变化选中时可加 hover 灰选中时容器变 #F2F4F7淡灰(+ icon 红方块 + 文字红色，3 处叠加)
营销态 / icon 态—（通用 tabbar 不区分）圆形位图插槽(注入频道头像 / 商品营销图)
招手位同 video 场域红点 / 数字 / 文字 3 类 badge（规格继承通用）

1.2 行为准则

  1. 视频场域选中态 = 3 处叠加:容器变淡灰 + icon 单色 → 红白双色 + 文字色黑→红 + 字重加粗 — 详 ch-2 § 选中态
  2. 选中态没有「红方块外框」独立元素 —— 红方块就是 video icon 本身在选中态的形状(同 icon 的 line/filled 双态),详 ch-2 § 视觉解剖
  3. icon 态 / 营销态 = 圆形位图插槽,业务方注入位图(频道头像 / 用户头像 / 商品营销图)
  4. 招手 badge 红色统一用 color_primary(#FF0F23)；video 实心图标外层 #FF0000 需设计侧确认是否并入主色
  5. 所有视觉规则继承通用 tabbar:坑位数 2~5、文案最多 4 字、icon 20×20、安全区 17 DP 等,见 通用 tabbar ch-1 行为准则

V16 立场:本组件作为视频场域的专属 atom,只允许在视频/逛频道相关页面使用;不允许把红色 video icon 用在非视频频道的选中态(会让用户误判频道身份)。

2原子结构

👉 一句话拆解:每个 44×44 的坑位由 3 部分组成:

  • ① 图标(20×20):默认是 video 播放图标(线性) / 选中变红实心 / 营销/icon 态变圆形位图头像
  • ② 文字(下方):「逛逛」/「文案」,默认黑色,选中变红色加粗
  • ③ 招手位(右上角,可选):红点 / 数字 / 文字 三选一

状态切换:默认/选中/icon/营销 共 4 类。选中态跟其他三态最大差异是「icon 形状变红实心 + 文字变红」—— 这是视频频道的视觉印记。

完整 token / DP / 字号字重表,见专业版。

3变体矩阵

👉 一图看懂:总共 12 个变体,按「底导类型 × 招手类型」二维组合。视觉上分 4 大类:

  • ① 默认态 × 4 招手 (4 个) — 黑色 video icon + 黑字「逛逛」
  • ② icon 态 × 2 招手 (2 个) — 圆形头像位图,无文字
  • ③ 选中态 × 4 招手 (4 个) — 红色 video icon + 红字 Semibold
  • ④ 选中态-X × 3 招手 (3 个,命名冗余) — 视觉同 ③

⚠️ 属性名残留:营销态数字 / 文字在 Relay 中仍写作「选中」，选中派生写作「状态=选中态」。文档按结构归类，消费时以视觉结构为准。

3.3 Joy Agent 头像 4 状态（52×52）

4多端适配 / 应用场景

👉 用在哪:本 atom 不直接渲染,作为通用 tabbar的 slot 内容,在视频场域两种业务场景使用:

  • 🌞 双列商品瀑布流场景 → 跟视频场域 default light 形态 tabbar 一起用
  • 🌑 沉浸式视频播放场景 → 跟视频场域 default dark 形态(Liquid Glass) tabbar 一起用

两种场景下 slot atom 规则一致，差异来自父级 tabbar 容器、是否挂 Joy Agent，以及 light / dark 材质。

本 atom 在 3 形态 × 2 场景 = 6 个组合下的实际渲染（含 5 个坑位的整条 tabbar）。视觉上注意:atom 本身（单坑视觉规则）6 个组合完全一致,差异在父级 tabbar 容器(default vs joy-agent 形态)、Joy Agent 状态(默认/招手)、和 mode(light vs dark)。

4.3 反例

❌ 不允许:把红色 video icon 用在非视频频道(如把首页 home icon 在选中态变成红 video icon)。这会让用户误判频道身份。

❌ 不允许:在 dark mode 下把选中态文案改成其他色(如白色/黄色)。#FF0F23 红色是视频频道的视觉印记,穿透 light/dark。

❌ 不允许:把 icon/营销态 圆形位图换成方形 / 圆角矩形。r=100 圆形是视频场域 atom 的稿件约定,跨业务保持视觉一致。
