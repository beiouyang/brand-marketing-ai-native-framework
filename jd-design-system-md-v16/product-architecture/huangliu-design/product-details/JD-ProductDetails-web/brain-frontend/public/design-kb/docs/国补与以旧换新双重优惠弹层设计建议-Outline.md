# 设计建议 Outline — 国补&以旧换新双重优惠弹层

> 生成时间：2026-05-27

---

## 一、需求解读

**要解决的问题：** 用户在优惠弹层中看到国补和以旧换新两项优惠分散展示，无法直观感知叠加后的总优惠金额，需要自行心算组合。

**业务目标：** 通过优惠弹层的设计优化，强化用户对双重优惠的感知，降低用户决策成本，促进订单转化。

**设计挑战：** 在弹层的有限空间内，既要分别展示两项优惠的独立信息（各自金额、条件、有效期），又要聚合突出叠加后的最终到手价；同时需处理多种状态组合（国补已领/未领、有旧机/无旧机）而不互相混淆。

**成功标准：** 用户打开优惠弹层，一眼就能看到「国补减 xxx + 以旧换新再减 xxx = 到手价 xxx」的叠加结论，不需要拼接认知。

---

## 二、初步设计判断

1. **分行展示各项优惠，但优化排序逻辑** — 国补和以旧换新不应平等对待。优惠弹层应优先突出「国家补贴」这一政策性优惠（独占一个较大的视觉区块），以旧换新作为配角补充其下；这样的信息层级符合政策重要性和用户决策权重。

2. **聚合展示是弹层顶部的「核心结论区」，不是底部补充** — 不宜把「国补+以旧换新=xxx」作为小文案贴在底部。应在弹层顶部设置一个独立的「叠加利益点汇总卡片」，用数字 + 颜色对比突出 xxx，让用户第一眼落在聚合结论，然后往下看详情。

3. **用颜色体系区分两项优惠的语义** — 国补用绿色（#0AAD48，政策属性），以旧换新用服务金色（#b5691a，权益属性），叠加结论用黑色/主题色（综合权益），三种颜色的搭配避免视觉混乱。

4. **处理「已领 vs 未领」的状态转换** — 当国补未领取时，弹层内该行应展示「国补可减 xxx，点击领取」，而非直接显示减额。这要求弹层的国补优惠行具备「动态转换」能力，领取后实时变为「已减 xxx」，无需重新打开弹层。

5. **以旧换新有旧机 / 无旧机的两套数字展示** — 有旧机时显示「换新到手价」（已确定），无旧机时显示「换新预估价」（待选择）。弹层需同步这两套数字的切换，避免用户误认为「预估价」就是最终结果。

## 三、动笔前需要确认的问题

| # | 问题 | 不确认的后果 |
|---|---|---|
| 1 | **两项优惠都满足时，弹层是否需要展示「不满足」情况？** 例如用户无旧机时，以旧换新行是否隐藏或降级显示？ | 如果设计了「预估价」状态但研发简化为隐藏，用户会错误认为只有国补一项优惠 |
| 2 | **国补未领取时，用户在弹层中点击「领取」后，弹层中的数字是否实时更新？** 还是需要关闭重新打开？ | 决定弹层的交互复杂度和技术成本 |
| 3 | **横滑容器是否需要支持？** 如果优惠条目过多（国补/满减/支付立减/政府券等叠加），是否允许横滑？ | 影响弹层的宽度设计和多条优惠的排列策略 |
| 4 | **聚合结论中「到手价」是否需要展示价格锚点？** 如「近90天最低」或「节省 xxx」的对比？ | 决定聚合卡片的信息密度和用户对价格力的感知 |
| 5 | **线下门店降级时，两项优惠中哪一项应该隐藏或降级？** | 影响门店用户的权益感知和交互流程 |

---

## 四、需要覆盖的状态

- ⭐ **国补已领 + 以旧换新有旧机** → 显示「国补再减 xxx + 以旧换新再减 xxx = 到手价 xxx」（双项已确定）
- ⭐ **国补已领 + 以旧换新无旧机** → 显示「国补再减 xxx + 以旧换新预估 xxx」（一项已确定，一项待确认）
- ⭐ **国补未领 + 以旧换新有旧机** → 显示「国补可减 xxx（领取入口）+ 以旧换新到手价 xxx」（一项需引导，一项已确定）
- 国补未领 + 以旧换新无旧机 → 两项都是预估/待确认状态
- ⚠️ **无旧机选择时，弹层的以旧换新行是否保留占位？** 还是完全隐藏？需确认产品策略
