# 国补资格弹层换绑场景文案兼容小屏 · 设计决策记录

> 来源：设计稿 [【视觉定稿】国补资格弹层换绑场景文案兼容小屏](https://relay.jd.com/file/design?id=2046484439810203649&page_id=6:72123&node_id=6:72153) / PRD 【同省不同市活动的换绑优化】
> 设计：林梦笔（UI/UE）/ 产品：范育宇、马兰 / 时间：2025-09-09

---

## 一、需求背景

**核心问题（来自 PRD §1.1 + 设计稿背景标注）：**

两个并行问题合并在同一 PRD 中：

1. **同省不同市地址展示缺陷**：系统仅展示到省份层级，未细化到市级。当用户在同省份内进行跨市换绑操作时，无法直观区分具体目标城市，可能引发误操作。

2. **换绑弹层文案规范不清晰 + 小屏适配缺失**：目前在商详领取弹层内，已在其他平台绑定资格时文案规范不清晰，且未做小屏适配，导致小屏（320px）展示字体模糊、间距错位。

**核心痛点及目标：**
- 同省跨市换绑场景升级地址展示精度至市级，消除误操作风险
- 规范化「被其他平台占用」场景的解绑引导文案，降低用户决策成本，对齐竞品（淘宝/拼多多已有明确引导）
- 适配 320px 小屏，修复布局错乱问题

> 来源：PRD §1.1 / 设计稿背景标注

## 二、设计核心打法

> 这一章记录本次需求的关键设计举措——围绕核心动作深思熟虑的部分。

### 核心举措

**这次需求的核心动作是：** 在换绑弹层内，将「已被其他平台占用」的纯告知文案升级为带平台名的精准解绑引导文案，同时实现 375/320 双屏宽自适应布局，修复小屏错位问题。

**为什么这样做：**
- 竞品对标（来自 PRD）：淘宝/拼多多等平台被占用时有具体 todo 文案告知引导解绑行为；京东仅展示「被其他平台/地区领取」，没有引导解绑文案，用户茫然无措。
- 小屏问题（来自设计稿背景标注）：320px 机型上文案因未做适配导致字体模糊、间距错位，需要修复。

**排除的方案：**（PRD/设计稿中无明确记录）

**对其他模块的影响：**
- 换绑区块文案：灰色改为「已领资格当前地址不可用」，绿色展示二级市「当前地址资格可用」
- 解绑领取弹层：被其他平台占用时，原有文案追加「请去相关平台解绑后再来领取」
- 弹层整体布局：375/320 双屏宽需分别出图，实现自适应切图

### 同省不同市地址展示规则（来自 PRD §2）

| 场景 | 旧展示方式 | 新展示方式 |
|---|---|---|
| 跨省场景 | XX省→XX省 | 维持不变 |
| 同省跨市场景 | XX省→XX省 | 升级为 XX省XX市→XX省XX市 |

- 展示地址按活动维度走，取活动举办地区字段
- 后端传一级/二级地址，前端优先取 URL 拼接的省市参数，无法获取则按后端传入展示
- 展示到二级地址时，前端需要调整字体大小

> 来源：PRD §2

### 是否有多套方案

- [x] 有多套方案并行输出（375/320 × 白天/暗黑，共 4 组）

**多套方案说明：**

| 方案 | 核心差异 |
|---|---|
| AFTER 375标准屏适配 | 规范化文案 + 正确布局（白天） |
| AFTER 320小屏适配 | 收窄自适应版本（白天） |
| AFTER 375标准屏 暗黑适配 | 暗黑模式 |

## 四、交互流程及逻辑判断

> 模块之间的点击跳转、展开/收起等交互逻辑。

**主流程：**
商详主页 → 点击国补入口 → 进入领取弹层 → 触发「已被其他平台占用」场景 → 展示带引导的解绑文案 → 用户点击解绑链接/了解如何解绑 → 前往对应平台解绑 → 返回京东重新领取

**分支逻辑：**

| 触发条件 | 动作 | 结果 |
|---|---|---|
| 换绑场景·同省跨市 | 展示二级市地址（省+市） | 地址精度提升，用户可明确区分 |
| 换绑场景·当前地址可用 | 绿色展示二级地址 + 可用标识 | 正向引导换绑 |
| 换绑场景·当前地址不可用 | 灰色展示不可用文案 | 告知用户当前地址无法享受补贴 |
| 资格被其他平台占用·平台可识别 | 展示带平台名的精准引导文案 | 用户明确知道去哪个平台解绑 |
| 资格被其他平台占用·平台不可识别 | 展示通用引导文案（其他平台） | 用户知道需要去其他平台解绑 |
| 屏幕宽度 = 320px | 弹层自适应收窄布局 | 文字/间距正常显示，无模糊 |

## 附：可复用的设计模式

### Pattern 1：被第三方平台占用时的精准引导文案结构

**适用场景：** 用户的某项权益/资格已在其他平台绑定，需要引导用户去对应平台解绑后再回来操作。

**不适用场景：** 资格被同平台占用（京东内部占用），无需引导去第三方。

**核心原则：** 平台可识别时，直接命名平台（「去拼多多解绑」优于「去其他平台解绑」），降低用户的认知跳转成本。

### Pattern 2：双屏宽自适应文字容器

**适用场景：** 弹层/卡片内存在动态长度文案（如含地区名、平台名的拼接文案），需要同时适配 375px 和 320px 屏宽。

**不适用场景：** 固定短文案（如纯按钮标签），不需要自适应处理。

**核心原则：** 设计侧出「自适应伸缩区域」标注，研发按弹性布局实现；不要为每种屏宽单独写死文字换行位置。
