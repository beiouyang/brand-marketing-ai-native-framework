## 0. 需求背景

### 用户问题
App 商详页在国补与以旧换新同时可用时，用户难以感知两项叠加后的到手价优惠，导致利益点传达不充分。同时，用户在不同国补资格状态下（无资格 / 有资格未领取 / 已领取）看到的页面信息缺乏明确区分，无法快速判断自己处于哪个阶段、还需做什么。

### 根本原因
- 国补资格状态直接影响用户决策路径，原有设计未针对三个状态做差异化处理
- 国补与以旧换新作为两个独立促销体系，叠加后利益点在页面上未形成聚合展示，用户需分别感知

---

## 1. 设计目标与边界

**目标：** 通过对国补三态（无资格 / 有资格未领取 / 已领取）的差异化视觉设计，并在已领取态下突出呈现国补叠加以旧换新的到手价利益点，提升用户对总优惠的感知，促进领取转化和下单决策。

**明确不做的事：**
- 不修改国补资格的判断逻辑（由后端业务方决定）
- 不重新设计以旧换新的选机 / 估价流程（见 `pdp/trade-in/app-trade-in-v15-reasoning.md`）
- 不覆盖结算页的国补展示逻辑

---

## 2. 约束条件

| 类型 | 约束内容 |
|---|---|
| 技术约束 | 国补资格状态（无资格 / 未领取 / 已领取）由后端接口下发，前端根据状态切换展示模块；未添加旧机时展示最高优惠金额（支付击穿场景） |
| 业务约束 | 国补为国家政策，叠加规则（国补 + 以旧换新是否可同时享受）由政策规定；线下门店需降级展示 |
| 设计系统约束 | 延用服务金色（ServiceGolden）体系：`#ffd4a3`（背景）/ `#b5691a`（文字）/ `#994d00`（深色），用于楼层加盖；国补专属绿色 `#0AAD48` 延续 App 国补多商品方案的语义 |
| 平台约束 | App 端，375px 宽度；状态切换通过半弹层操作，不触发页面跳转 |

## 3. Before → After

| 问题（Before） | 设计决策 | 结果（After） |
|---|---|---|
| 国补三种资格状态视觉无区分，用户不清楚自己处于哪个阶段 | 为无资格 / 有资格未领取 / 已领取三态设计差异化的价格楼层与腰带展示 | 用户可即时识别当前国补状态，有资格时有明确领取引导 |
| 国补叠加以旧换新时，两项优惠分散在不同楼层，无聚合呈现 | 在已领取态下，将国补 + 以旧换新叠加后的到手价统一呈现在核心利益点位置 | 用户一眼看到两项叠加的最终优惠金额，减少认知拼接成本 |

## 4. 核心设计决策

### Decision 1：以国补三态为核心分支，驱动价格楼层和腰带的差异化展示

**决策：** 将国补资格分为「无资格」「有资格未领取」「资格已领取」三个独立状态，每个状态对应不同的价格楼层展示方案和腰带内容，通过视觉语言明确告知用户当前所处阶段。

**原因：**
- 三种状态的用户目标完全不同：无资格用户需要知道「为什么没有」、未领取用户需要被引导「去领取」、已领取用户需要确认「我已拿到了」
- 统一展示会让不同状态的用户都感到信息不准确或行动指引模糊
- 状态驱动的模块切换在工程上可以由后端字段控制，前端只需按状态渲染对应组件

**对其他组件的影响：** 价格楼层、国补腰带、以旧换新弹层均需支持三态切换。

---

### Decision 2：有资格未领取态——强化领取引导，区别于已领取态的「确认感」

**决策：** 「有资格未领取」状态下，腰带采用更强的行动引导样式（带箭头/按钮入口），文案使用「最高减 xxx」表达潜在利益上限；「已领取」状态下，按钮消失，文案切换为「再减 xxx」，传达已到手确定感。此设计延续自 `app-national-subsidy-multisku-reasoning.md` Decision 2，并扩展为覆盖三态。

**原因：**
- 「最高减」= 潜在上限，强调值得去领取；「再减」= 确定减额，强调已到手
- 领取后移除引导按钮，避免已领取用户产生「还需要再做什么」的困惑
- 与多商品方案保持一致，降低历史 Case 的认知迁移成本

---

### Decision 3：已领取态下，国补叠加以旧换新利益点聚合展示

**决策：** 在用户已领取国补资格且商品支持以旧换新时，在价格楼层区域聚合展示「国补减 xxx + 以旧换新再减 xxx = 叠加到手价 xxx」的利益点组合。两项优惠以视觉区块的形式并排或层叠展示，颜色使用服务金（ServiceGolden）体系区分于纯国补的绿色语义。

**原因：**
- 用户已完成领取行为，此时的核心诉求从「引导领取」转为「确认总优惠」
- 将两项拆散展示需要用户自行心算叠加，聚合展示直接给出结论，降低决策成本
- 服务金色（#b5691a / #ffd4a3）用于「服务类权益」的视觉标识，与国补专属绿色语义独立，避免混淆

## 8. 可复用的设计模式

### Pattern 1：三态状态机驱动的促销模块切换

**适用场景：**
- 同一商详页面因用户状态不同（未参与 / 可参与 / 已参与）需要展示完全不同的促销内容
- 状态由后端接口下发，前端按状态渲染对应组件

**触发结构：**
1. 后端返回用户状态字段（0 = 无资格 / 1 = 有资格未领取 / 2 = 已领取）
2. 前端对应渲染三套腰带 + 价格楼层模板
3. 用户完成领取后，页面无需刷新，由前端状态变更触发局部更新

**不适用场景：**
- 状态差异仅体现在文案层面（无需换组件，直接改文案即可）
- 超过 4 种状态的复杂场景（建议拆分为多个独立需求）

**核心原则：** 状态机切换的边界要清晰，每种状态对应唯一的视觉方案，禁止状态之间的视觉混用。

---

### Pattern 2：多优惠聚合呈现——叠加利益点的感知提升

**适用场景：**
- 用户同时享有多项独立优惠（国补 + 以旧换新 / 平台券 + 活动折扣等）
- 单项优惠金额较小，但叠加后总优惠金额具有显著吸引力

**触发结构：**
1. 确认用户已满足所有优惠条件（均已激活 / 领取）
2. 在核心利益点位置聚合展示「项目A减xxx + 项目B减xxx = 到手价xxx」
3. 详情通过以旧换新弹层按需展开，主页只展示聚合结论

**不适用场景：**
- 用户仅满足部分优惠条件（未满足的优惠不应计入聚合展示）
- 优惠之间存在互斥规则（见 `app-trade-in-v15-reasoning.md` Pattern 2）

**核心原则：** 聚合展示的是「已确定可拿到」的到手价，不是「最高可能」的估算值；估算值应用「最高减」表达，确定值用「再减」或「到手价」表达。
