# 商详店铺 1+1 素材审核与预览规范

## 1. 适用范围

本文档用于商详店铺 1+1 页面素材上传、审核与预览渲染规则说明。

适用素材包括：

- 图1：系统底图
- 图2：商家沉浸式氛围背景图
- 图3：商家核心视觉元素图
- 图4：系统前景装饰图
- 图5：系统顶部装饰图
- Logo
- 品牌店铺标题切图

审核负责发现素材问题并给出修改建议；预览负责真实还原商家上传素材的最终展示效果。审核与预览独立执行，预览不自动修正素材。

以下素材由系统直接读取，不允许商家替换：

| 图5 | 系统顶部装饰图 | 固定叠加在所有系统图层最上方 |
| 品牌店铺标题切图 | Logo右侧标题 | 与Logo作为整体展示 |

## 5. 画布与坐标

所有尺寸和坐标均基于 3 倍图设计稿。

当前预览画布：

```text
1125 × 2439px
```

预览区缩放时，只允许整体画布缩放，不允许因容器缩放导致图层相对位置错位。

图1作为 `relative` 容器，图2、图3、图4、图5采用 `absolute` 定位。

推荐上传：

1125 × 993px（3倍图）

审核检查：

- 尺寸是否为 `1125 × 993px`

不再检查文件大小。

图2固定展示于图1指定区域：

| 属性 | 数值 |

| 展示高度 | 621px |

| 裁切方式 | 等比缩放，居中裁切 |

## 7. 图3 核心视觉元素图

### 7.1 商家上传提示

页面展示文案：

```text
建议上传728×387px透明底图片
```

推荐上传：

- `728 × 387px`（3倍图）
- PNG透明底

### 7.2 审核规则

图3审核仅检查：

- 是否为透明底 PNG
- 主体占比是否过小

不再检查：

- 左侧留白是否小于等于 10px
- 右侧留白是否小于等于 10px
- 固定边距
- 尺寸范围拦截
- 拉伸变形

### 7.3 主体占比异常

以下情况视为主体占比过小：

- 人物或核心元素明显偏小
- 大量区域为空白
- 主体未充分利用画布空间

### 7.4 预览规则

审核发现问题时，预览区仍使用商家原始上传素材。

禁止：

- 自动裁切透明区域
- 自动放大主体
- 自动修改位置
- 自动调整比例
- 自动修正素材

预览区需保证商家看到的效果与最终线上实际效果一致。

### 7.5 隐藏安全展示区域

安全展示区域仅用于布局计算，不向商家展示边框、辅助线或尺寸标注。

3倍图尺寸：

| 属性 | 数值 |
| --- | --- |
| 宽度 | 522px |
| 高度 | 312px |
| 距离顶部 | 1104px |
| 距离右侧 | 39px |
| 距离底部 | 1023px |

图3预览按商家上传素材的原始画布适配安全展示区域宽度，保持原始比例并居右展示；透明留白会真实显示，不做裁切补偿。

## 8. Logo

### 8.1 推荐上传规范

Logo推荐上传：

- PNG透明底
- Logo主体清晰完整
- Logo主体应尽量撑满画布

### 8.2 审核规则

Logo审核仅检查：

- 是否为透明底 PNG
- Logo主体占比是否过小
- 深浅适配是否符合当前背景

不检查：

- 上传素材实际宽度
- Logo是否大于 130px
- Logo是否小于 130px
- 左右边距像素值
- Logo模糊
- 深色背景对比度数值

### 8.3 深浅适配规则

系统根据 Logo 所在预览区域的背景明暗进行判断：

- 深色背景下，应使用反白 Logo
- 浅色背景下，应使用深色 Logo

不符合时提示：

```text
Logo深浅适配不符合
```

修改建议：

```text
请根据背景深浅上传适配版本：深色背景使用反白Logo，浅色背景使用深色Logo。
```

### 8.4 Logo预览展示规则

预览展示按线上展示规则自动适配。

默认情况：

- Logo高度：24px（1倍图）
- 3倍图展示高度：72px
- 宽度等比缩放
- 保持原始比例

超宽Logo：

当按 24px 高度缩放后：

```text
Logo宽度 > 130px
```

则：

- 宽度限制为 130px（1倍图）
- 3倍图最大宽度为 390px
- 高度按比例自动缩小
- 保持原始比例

方形Logo：

当：

```text
宽高比 ≤ 1.3 : 1
```

则：

- 自动放大展示
- 高度调整为 36px（1倍图）
- 3倍图展示高度为 108px
- 保持比例缩放
- 不向商家展示修改建议

### 8.6 Logo异常预览规则

即使审核发现问题，预览区仍展示商家上传素材的真实效果。

禁止：

- 自动裁切
- 自动修正
- 自动补偿
- 自动裁切透明留白
- 自动放大主体

留白过大、主体过小等问题仅在审核结果中提示。

## 12. 审核结果输出

审核结果不按单个素材分散输出，而是综合以下素材统一输出：

- 图2 沉浸式氛围背景图
- 图3 核心视觉元素图
- Logo

统一审核结果示例：

```text
审核通过
```

或：

```text
审核未通过
```

当存在问题时，需分别说明对应素材原因。

示例：

```text
问题：
- 背景图尺寸不符合要求
- 核心视觉元素图主体占比过小
- Logo深浅适配不符合

修改建议：
- 上传1125×993px背景图
- 优化核心视觉元素主体占比
- 深色背景使用反白Logo，浅色背景使用深色Logo
```

## 13. 单素材审核状态

三个上传区域下方都需要展示各自审核状态。

展示状态：

- 未上传：提示上传后显示审核结果
- 审核通过：展示审核通过
- 审核不通过：展示审核不通过，并列出错误项

## 14. 审核与预览原则

审核负责发现问题并给出修改建议。

预览负责真实还原素材效果。

两者独立执行：

- 审核可以提示问题
- 预览不自动修正素材

确保商家能够真实感知当前素材的最终线上展示效果。
