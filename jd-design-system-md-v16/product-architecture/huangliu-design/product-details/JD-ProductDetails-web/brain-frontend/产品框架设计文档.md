# 商详大脑 - 产品框架设计文档

## 一、项目背景与核心定位

### 1.1 项目愿景
将AI能力注入商品详情页设计全链路，通过**知识沉淀 + 智能工具**双线整合，打造设计师专属的智能工作中台。

### 1.2 核心定位
**「设计师的AI搭档」** — 不是替代设计师，而是增强设计师的能力：
- 够快速溯源历史决策与规范
- 能智能判断新需求的合理性
- 可自动化执行重复性工作
- 持续沉淀设计资产与经验

### 1.3 当前设计工作链路梳理

```
┌─────────────────────────────────────────────────────┐
│  需求承接线                                           │
│  PRD承接 → 需求评审 → 设计出稿 → 内部审核             │
│  → 开发落地 → 设计走查 → 上线跟进数据                  │
├─────────────────────────────────────────────────────┤
│  设计预研线                                           │
│  监测竞品 → 分析自身问题 → 寻找优化课题                │
│  → 产出方案 → 推进上线 → 跟进数据效果                  │
├─────────────────────────────────────────────────────┤
│  资源审核线                                           │
│  业务提供素材 → 审核合理性 → 预览配置效果               │
└─────────────────────────────────────────────────────┘
```

## 二、产品双线架构

```
                    商详大脑
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     【主线】      【分支】       【整合层】
     知识库沉淀    Skill工具链    AI协同工作台
          │            │            │
    ┌─────┴─────┐  ┌──┴──┐  ┌────┴────┐
    │历史需求    │  │素材 │  │智能问答  │
    │设计规范    │  │生成 │  │需求评审  │
    │模块逻辑    │  │     │  │竞品分析  │
    │数据效果    │  │智能 │  │方案建议  │
    │竞品信息    │  │走查 │  │持续沉淀  │
    └───────────┘  │     │  └─────────┘
                   │竞品 │
                   │分析 │
                   └─────┘
```

### 2.1 主线：知识库（Knowledge Base）
目标：让所有设计资产与经验「可溯源、可检索、可关联」。

| 知识类型 | 内容 | 用途 |
|---------|------|------|
| 模块设计规范 | 头图、价格区、评价区、推荐区等各模块的现状设计逻辑 | 新需求评审的依据 |
| 历史需求档案 | 需求PRD + 设计稿件 + 上线数据效果 （三位一体关联） | 历史决策溯源 |
| 变动记录 | 每个模块历次改动的版本记录、原因、效果 | 避免重复踩坑 |
| 竞品信息库 | 竞品商详页结构、功能、设计截图与分析 | 竞品研究数据源 |
| 数据看板 | 各模块核心指标历史趋势 | 设计效果量化 |

### 2.2 分支：Skill工具链（Tool Chain）
目标：覆盖生产前-中-后链路，自动化提效。

| 阶段 | Skill | 功能描述 |
|------|-------|---------|
| 🔵 产前 | 竞品收集+分析 | 自动抓取竞品商详截图，AI对比分析差异 |
| 🔵 产前 | 需求合理性判断 | 基于知识库，对新PRD进行合理性评估 |
| 🟢 产中 | 素材生成 | 基于规范和文案，自动生成设计素材 |
| 🟢 产中 | 规范校验 | 设计中实时校验是否符合规范 |
| 🟡 产后 | 智能走查 | 对比设计稿与线上页面，自动检测差异 |
| 🟡 产后 | 数据归因分析 | 上线后自动关联改动→数据变化 |

### 2.3 整合层：AI协同工作台
目标：两线整合，为设计师提供统一的AI工作界面。
- **查**：自然语言查询规范/历史/竞品
- **判**：协助判断需求合理性，给出建议
- **做**：调用Skill完成具体任务
- **存**：增量内容自动沉淀回知识库

## 三、MVP版本功能定义

### 3.1 MVP功能矩阵

```
优先级 P0（必须有）：
  ✅ AI对话助手         — 核心交互入口
  ✅ 知识库检索         — 规范、历史需求查询
  ✅ 模块浏览           — 按商详模块分类浏览规范
  ✅ 需求录入+关联      — 新增需求沉淀到知识库

优先级 P1（建议有）：
  ⬜ 竞品收集Skill      — 自动化竞品截图+分析
  ⬜ 智能走查Skill      — 设计稿vs线上对比
  ⬜ 需求评审辅助       — AI给出合理性评估

优先级 P2（后续迭代）：
  ⬜ 素材生成Skill      — 自动生成设计素材
  ⬜ 数据看板           — 模块指标可视化
  ⬜ 团队协作           — 多人共享知识库
```

### 3.2 MVP核心功能说明

| 功能模块 | 核心能力 | 用户场景举例 |
|---------|---------|------------|
| **AI对话** | 自然语言对话，跨知识库查询 | "头图区域最近改过什么？上次改动数据怎么样？" |
| **模块总览** | 商详各模块可视化浏览，进入查看规范和变动历史 | 点击「价格区」模块→看到规范卡片+变动时间轴 |
| **需求档案** | 历史需求列表，搜索、筛选、查看详情（PRD+稿件+数据） | 搜索"限时降价"→看到所有相关历史需求的完整链路 |
| **需求录入** | 表单/对话引导录入新需求，自动关联到对应模块 | 输入PRD→AI提取关键信息→自动归类到对应模块 |
| **知识图谱** | 需求-模块-数据 三者的关联可视化 | 看一个模块时能看到所有关联的需求和数据变化 |

## 五、交互形式设计

### 5.1 三种核心交互模式

| 模式 | 描述 | 适用场景 | MVP实现 |
|------|------|---------|---------|
| **对话式 (Chat)** | AI对话助手，自然语言驱动 | 查询、建议、评审 | ✅ P0 |
| **浏览式 (Browse)** | 结构化浏览知识库内容 | 查看规范、历史需求 | ✅ P0 |
| **工具式 (Tool)** | 参数化工具面板，一键执行 | 走查、生成、分析 | ⬜ P1 |

### 5.2 对话交互设计

```
┌──────────────────────────────────────────────┐
│  💬 商详大脑 AI 助手                          │
│  ┌──────────────────────────────────────────┐│
│  │  当前上下文：价格区模块  |  相关需求: 2个  ││
│  ├──────────────────────────────────────────┤│
│  │                                          ││
│  │  👤 "价格区最近一次改了什么？效果怎么样？"    ││
│  │                                          ││
│  │  🤖 价格区最近一次改动是在 2024.11.15，    ││
│  │     主要变更：                            ││
│  │     ┌─────────────────────────────────┐   ││
│  │     │ 📄 需求 #128：限时降价标签优化    │   ││
│  │     │ 变更内容：新增"限时直降"标签样式  │   ││
│  │     │ 上线日期：2024.11.15             │   ││
│  │     │ 数据效果：点击率 +12% → 查看详情  │   ││
│  │     └─────────────────────────────────┘   ││
│  │     ...                                  ││
│  │                                          ││
│  ├──────────────────────────────────────────┤│
│  │  /查询规范  /评审需求  /分析竞品  /录入需求  ││
│  ├──────────────────────────────────────────┤│
│  │  💬 输入你的问题...                  📎 →  ││
│  └──────────────────────────────────────────┘│
└──────────────────────────────────────────────┘
```

**AI回复卡片类型：**
- **需求卡片**：标题、状态、模块、数据摘要、详情入口
- **规范卡片**：模块名、设计规则摘要、关联变动
- **竞品卡片**：截图对比、差异分析、参考价值评级
- **评审卡片**：合理性评分、风险提示、历史参考、建议方案

### 5.3 浏览交互设计

**模块总览 - 交互规则：**
- 卡片网格展示所有商详模块（头图、价格、评价...）
- 点击卡片进入模块详情 → 左侧规范说明 / 右侧变动时间轴
- 支持搜索/筛选

**需求档案 - 交互规则：**
- 默认表格视图展示需求列表（可切换卡片视图）
- 点击行/卡片进入需求详情抽屉或全屏页
- 详情页：Tab切换 PRD摘要 / 设计稿 / 数据效果 / 关联需求
- 强大的搜索+筛选（模块、状态、时间、关键词）

### 5.4 快捷指令系统

在AI对话中，输入 `/` 触发快捷指令菜单：

| 指令 | 功能 |
|------|------|
| `/查询规范 [模块名]` | 快速查询某模块的设计规范 |
| `/评审需求` | 以对话方式提交新需求做合理性评审 |
| `/分析竞品 [竞品名]` | 调取竞品数据库中的信息做对比 |
| `/录入需求` | 引导式录入新需求并关联模块 |
| `/走查对比` | 上传设计稿，AI对比线上页面 |
| `/本周动态` | 查看本周商详模块的变动汇总 |
