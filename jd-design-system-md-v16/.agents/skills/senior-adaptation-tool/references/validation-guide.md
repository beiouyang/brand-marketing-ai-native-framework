# 适配结果验证指南（通用）

适用于任意 Zero 设计稿的大字（1.15x）/ 长辈版（1.3x）适配结果验证，不与具体页面绑定。

## 验证清单

### 0. 前置门禁验证
- [ ] 已检查当前会话是否存在 zero-design MCP
- [ ] 已检查用户是否指定了当前 Zero 文件中的原画板（节点 ID 或名称）
- [ ] 若任一未满足，已明确提示"技能不适用"并终止执行
- [ ] 若都满足，继续后续适配与回写流程

### 1. 通用画板验证
- [ ] 原画板未被修改（位置、尺寸、内容均与原始一致）
- [ ] 新画板创建于原画板右侧
- [ ] 新画板命名规范：`<原画板名>-大字版`、`<原画板名>-长辈版`
- [ ] INSTANCE 处理（V2.2）：detach 在预处理画板统一执行一次（自底向上 + 出错回滚）；紊乱时降级为「跳过 detach、直接 rescale」
- [ ] 状态栏 / 导航栏未被缩放，保持原始尺寸
- [ ] **textAutoResize 智能选择（V2.1）**：原 WIDTH_AND_HEIGHT 保持、原 HEIGHT 保持、原 NONE 改 HEIGHT；**未把 HEIGHT 强行改为 WIDTH_AND_HEIGHT**
- [ ] 文字节点 `lineHeight = { value: Math.round(当前字号 × 1.2), unit: 'PIXELS' }`，**未使用 AUTO**
- [ ] 文字节点 `paragraphSpacing = 0` 且 `paragraphIndent = 0`
- [ ] **字号上限（V2.1）**：≥24pt（大字版）/ ≥26pt（长辈版）的常规 TEXT 已保持原值，未继续放大
- [ ] "有色标签"识别命中后整体 rescale，背景与文字未发生错位
- [ ] "有色标签"父容器联动放大严格遵守约束 A/B（仅紧贴 wrapper 才联动，复杂卡片不联动）
- [ ] 已缩放集合中的 TEXT 未被二次放大

### 2. 大字版验证（1.15x，仅文字放大 + 标签 visited 不 rescale，允许纵向延展）
- [ ] 命中"有色标签"识别的容器**仅 visited，未执行 rescale(1.15)**（与 iOS 系统大字版对齐）
- [ ] 未命中标签的 TEXT 字号已按字号上限规则处理：
  - ≥ 24pt → 保持原值
  - < 24pt → `Math.max(12, Math.round(原字号 × 1.15))`（四舍五入，最小 12px）
- [ ] 行高已与新字号同步（= `Math.round(新字号 × 1.2)` 固定像素值），段距为 0
- [ ] textAutoResize 已按 V2.1 智能选择规则处理（HEIGHT 保留、NONE 改 HEIGHT）
- [ ] 混合字号节点已逐段正确处理（fontSize + lineHeight + paragraphSpacing）
- [ ] 非文字非标签结构（图片、图标、纯容器）未被改动
- [ ] 文字与周围控件的上、下、左、右间距保持不变（**大字版严禁修改 padding/itemSpacing**）
- [ ] **大字版未执行 AutoLayout 间距同步**（V2.1 4.0.6 规则已在 V2.2 废弃，双版本均不动 padding/itemSpacing）
- [ ] 画板宽度与原画板一致；高度允许按 `max(y+height)+SAFE_PAD` 主动拉长

### 3. 长辈版验证（1.3x，V2.2 用户标注 + 标签整体缩放 + 不动间距）
- [ ] 已与用户确认组件分类清单（特殊处理类 / 常规处理类 / **V2.2 用户指定不放大类**）
- [ ] **V2.2 用户指定不放大节点**已识别并 visited（banner 横幅 / 排名徽章 / 店铺LOGO 等），保持原尺寸
- [ ] 顶部头部 banner 未被列入"特殊处理"（避免整体 1.3x 后超出 375 宽度被裁）
- [ ] "特殊处理"组件已整体 `rescale(1.3)`；子孙 TEXT 未二次放大
- [ ] **图文徽章识别（4.0.5）**：先于标签识别执行，命中节点已整体 rescale(1.3)
- [ ] "有色标签"自动识别命中节点已整体 `rescale(1.3)`，与"特殊处理"和"图文徽章"集合互斥
- [ ] "常规处理"组件中 TEXT 字号已按字号上限规则处理：
  - ≥ 26pt → 保持原值
  - < 26pt → `Math.max(12, Math.round(原字号 × 1.3))`
- [ ] 行高 = `Math.round(当前字号 × 1.2)`（固定像素），段距 = 0
- [ ] **V2.2 长辈版未执行 padding / itemSpacing 同步**（V2.1 4.0.6 规则已废弃，padding 本身不需要变）
- [ ] 画板宽度与原画板一致
- [ ] Auto Layout 祖先 `layoutSizingHorizontal = 'FIXED'`、`layoutSizingVertical = 'HUG'`（已排除 root 自身）
- [ ] HUG 仅作用于 Auto Layout FRAME；未对 GROUP / 非 Auto Layout FRAME 设 HUG
- [ ] 根 Frame 按 `max(y+height)+SAFE_PAD` 主动拉长
- [ ] 画板未发生横向扩展

### 4. 特殊字号 / ICON / 弹窗 验证
- [ ] 等级标识字母（LV.L/M/S）未被放大
- [ ] ¥ 与价格数字使用相同字号，未出现"符号悬浮"
- [ ] 关联型 ICON 已跟随文字放大；独立型 ICON（导航栏图标、Logo、关闭按钮）保持原尺寸
- [ ] 弹窗 / 浮层外框宽度严格保持，仅纵向 HUG

### 5. 回写验证
- [ ] 内容已正确绘制到对应新画板
- [ ] 层、命名、可编辑性保持完整
- [ ] 节点 ID 已结构化返回，便于后续引用

### 6. 截图校验（仅在运行环境支持图像识别时执行；否则跳过）
- [ ] 已截取原画板、大字版、长辈版三张截图
- [ ] 标签整体形态完好（背景 + 文字同步等比放大）
- [ ] 没有出现纵向裁切（横向溢出允许）
- [ ] 行间距视觉紧凑、无明显空行
- [ ] 整体结构与原稿一致、未出现模块顺序变化或重叠
- [ ] 未通过项已以"问题清单 + 节点 ID + 建议处理"形式输出，未尝试自动重做

### 7. 画板尺寸回归（仅在截图校验通过后执行）
- [ ] 新画板根 Frame 已设 `clipsContent = true`
- [ ] 新画板根 Frame 已 `resize(375, 812)`
- [ ] 锚点保持左上角（root.x、root.y 未变）
- [ ] 内部子节点的 x / y / width / height / rescale 均未被回缩或修改
- [ ] 截图校验未通过 / 环境不支持图像识别时，未执行此步骤（保留延展后的画板）

## 验证工具使用

### get_screenshot
对原画板与新画板分别截图，进行视觉对比，重点检查：
- 是否有文字裁切、重叠
- 特殊处理组件是否完整、不变形
- 间距是否保持

### get_design_metadata
检查节点的字号、位置、尺寸、层级是否符合预期。

### get_variables
检查文字样式变量（如有），确认字号值符合预期。

## 常见问题与解决

### 问题 1：文字被二次缩放
- **原因**：特殊处理组件或标签内的文字被常规流程再次放大。
- **解决**：检查"已缩放集合"是否在第一阶段完整登记其所有子孙 TEXT 节点；第二阶段必须跳过该集合。

### 问题 2：画板高度爆炸（>原始 2 倍）
- **原因**：标签父容器联动放大失控（约束 A/B 不严格），或 HUG 误作用于 GROUP。
- **解决**：回滚到"仅放大标签自身、不联动父容器"模式；HUG 仅作用于 Auto Layout FRAME。

### 问题 3：组件分类信息缺失
- **原因**：未询问或用户未答复组件分类。
- **解决**：按 SKILL.md 3.3 节标准话术询问，复述确认后再继续。

### 问题 4：字号低于 12px
- **原因**：原始字号过小，缩放后仍低于 12px。
- **解决**：强制最小字号 12px：`Math.max(12, Math.round(原字号 * scale))`。

### 问题 5：间距被压缩或重叠
- **原因**：文字放大后未保持其与周围控件的相对间距。
- **解决**：textAutoResize 按 V2.1 智能选择（不强制 WIDTH_AND_HEIGHT）+ 不动 padding/itemSpacing；如出现纵向重叠，依赖 4.0.3 HUG 自然撑开。

### 问题 6：原画板被修改
- **原因**：未先克隆而直接修改了原画板。
- **解决**：所有适配结果必须写入新画板，原画板严禁改动。

### 问题 7：Zero 回写失败
- **原因**：zero-design MCP 异常或参数错误。
- **解决**：重新检查 zero-design MCP 连接、文件 ID 是否匹配；不可谎报成功。

### 问题 8（V2.1）：商品标题挤穿价格、同行多文本互相覆盖
- **原因**：把原 `textAutoResize = 'HEIGHT'` 强行改为 `'WIDTH_AND_HEIGHT'`，导致长文本宽度无限膨胀挤穿邻居。
- **解决**：尊重原稿的 textAutoResize；HEIGHT 模式保留、NONE 改 HEIGHT、WIDTH_AND_HEIGHT 保留。详见 SKILL.md 4.0.2。

### 问题 9（V2.1）：装饰大标题溢出顶栏 / 标题区破坏视觉版式
- **原因**：原始字号 ≥ 24pt（大字版）或 ≥ 26pt（长辈版）的装饰性大标题继续被放大，撑出容器边界。
- **解决**：启用字号上限阈值，≥ 上限的字号保持原值不放大。详见 SKILL.md 2.5。

### 问题 10（V2.2 修订）：长辈版字大但呼吸感不变、文字之间挤压
- **原因**：误以为要靠放大 padding / itemSpacing 找回呼吸感。V2.1 曾追加 4.0.6 间距同步 ×1.3，**实测是错的**。
- **解决**：V2.2 已确认不动 padding / itemSpacing——文字字号变大撑大文字容器（HUG 自然撑高）后，间距相对内容自然合理。若仍觉拥挤，检查"有色标签 / 图文徽章 / 特殊处理组件"是否漏识别而未整体 rescale（呼吸感由它们的整体等比放大承担）。4.0.6 间距同步规则已废弃，**不要执行**。
