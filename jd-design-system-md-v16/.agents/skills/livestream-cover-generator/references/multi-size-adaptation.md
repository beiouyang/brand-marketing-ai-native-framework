# 多尺寸适配规则(7 商业化点位)

> 一张直播封面要适配 **7 个商业化点位**,不能各画各的。
> 本文件定义:**主输出 1:1 / 750×750px** + **如何派生 7 个点位的尺寸**。

## 7 个点位概览

| 点位 | 平台位置 | 用户场景 | 尺寸 / 比例 | 信息层处理 |
|------|---------|--------|----------|--------|
| **P1 · 首页** | 商详 / 首页推荐位 | 用户浏览首页 → 决定是否点 | 1:1(750×750)| 大字花字 + 主标 |
| **P2 · 频道** | 直播频道入口 | 用户进直播频道找节目 | 4:3(800×600)| 信息层简化,主体放大 |
| **P3 · 活动组件** | 大促活动页 / 频道 banner | 用户在活动页发现 | 16:9(960×540)| 横版,文字左 + 主体右 |
| **P4 · 预告聚合页** | 即将开播聚合 | 用户找未开播节目 | 1:1(750×750)| 标"预告" tag + 时间 |
| **P5 · 直播间-主播弹层** | 进直播间后弹窗 | 用户已在直播间 | 1:1(640×640)| 仅留主播 + 极简信息 |
| **P6 · 个人页-直播 Tab** | 用户主页直播分类 | 找特定主播节目 | 4:5(600×750)| 竖版,主播为主 |
| **P7 · 个人预告页** | 用户主页未开播 | 关注的主播未开 | 16:9(960×540)| 大主播头像 + 预告倒计时 |

## 派生规则:不重新生成 L1,只重排 L2 + L3 + L4

主输出(1:1 / 750×750)生成后,**复用 L1 背景**,只对 L2 + L3 + L4 做位置调整。

### 派生计算 · 通用规则

```
主输出 1:1 = (750, 750)

对每个点位 Px:
  ratio_p = width_p / height_p
  ratio_main = 1
  
  if ratio_p > ratio_main:   # 横版(更宽)
    crop_strategy = "center vertical, extend horizontal background"
    L4 reposition = "left or right block"
  elif ratio_p < ratio_main:  # 竖版(更高)
    crop_strategy = "center horizontal, extend vertical background"
    L4 reposition = "top or bottom block"
  else:                       # 1:1
    crop_strategy = "scale only"
    L4 reposition = "no change"
```

### Prompt 微调

派生时**不重跑** Generator A,只跑 **inpainting / outpainting**:

```text
[Original 750x750 cover],
Outpaint to [target_width]x[target_height],
Extend [direction: left/right/top/bottom],
Maintain [STYLE_KEYWORDS · from KB-A 调用],
Keep main subject at center,
Re-arrange text "[MARKETING_TEXT]" to [new position]
```

## 点位详细规则

### P1 · 首页 / 商详推荐位

| 项 | 值 |
|---|---|
| 尺寸 | 1:1 / 750×750 |
| 安全区 | 4 边各留 8% |
| 信息层 | 主标 + 副标双行 |
| 主体占比 | ≥ 50%(高视觉密度,争抢注意力)|

### P2 · 频道入口

| 项 | 值 |
|---|---|
| 尺寸 | 4:3 / 800×600 |
| 安全区 | 上 10% / 下 15%(预留 tab 切换栏) |
| 信息层 | 主标 only,副标可省 |
| 主体占比 | 60%-70%(频道内被多个并列展示,要更醒目) |

### P3 · 活动组件 / Banner

| 项 | 值 |
|---|---|
| 尺寸 | 16:9 / 960×540 |
| 安全区 | 左 / 右 15%(banner 可能被其他元素裁切) |
| 信息层 | 主标 + 副标 + CTA 按钮提示位 |
| 主体占比 | 40%-50%(横版,主体小,文字大) |

### P4 · 预告聚合

| 项 | 值 |
|---|---|
| 尺寸 | 1:1 / 750×750(与首页一致) |
| 安全区 | 同首页 + 顶部 5% 预留"预告"标签 |
| 信息层 | 加 **"预告 + 直播时间"** 模块 |
| 主体占比 | 40%-50%(被时间占用一部分注意力) |

### P5 · 直播间-主播弹层

| 项 | 值 |
|---|---|
| 尺寸 | 1:1 / 640×640(较小) |
| 安全区 | 4 边各 12%(底部弹层有 CTA 按钮) |
| 信息层 | **仅一行口号**(已在直播间,不用劝点) |
| 主体占比 | 80% 主播头像(直播间内对主播信任) |

### P6 · 个人页-直播 Tab

| 项 | 值 |
|---|---|
| 尺寸 | 4:5 / 600×750(竖版)|
| 安全区 | 上下各 8% |
| 信息层 | 主标在底部,主播头像在上 |
| 主体占比 | 70% 主播,纵向构图 |

### P7 · 个人预告

| 项 | 值 |
|---|---|
| 尺寸 | 16:9 / 960×540 |
| 安全区 | 右 30% 预留倒计时区 |
| 信息层 | 主标 + 倒计时(动态)|
| 主体占比 | 60% 主播,横版 |

## ⚠️ 边界

| 场景 | 处理 |
|------|------|
| 主输出主体居中,但 P3 横版需主体偏右 | Outpainting 时往左扩,主体相对右移 |
| 主输出文字在右,但 P6 竖版需文字在底 | 重排 L4,文字位置按构图 L-xx 末段规则 |
| L1 背景含强引导线(如放射线),被裁切后视觉残破 | 走 outpainting "complete the radiating pattern" |
| 平台某点位禁止某些视觉元素(如 P5 禁带价格)| Step 0 校验,失败则跳过该点位输出 |

## 效率建议

- **不必同时生成 7 个点位**,先输出 P1 主输出,**按需求生成其他点位**
- 灵创平台对接后,P2-P7 可走 batch outpainting(异步任务)
- 如果设计师手动调:推荐用 Figma plugin / 灵创设计稿模块在主输出基础上**派生**而非重画
