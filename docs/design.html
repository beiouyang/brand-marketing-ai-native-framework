典型场景示意

✅ 用：

  • 表单页提交 / 确认动作（下单确认、登录、支付、保存）
  • 页面级首要 CTA（立即购买、立即领取、加入购物车）
  • 模态 / 弹窗的主操作（确认删除、完成、知道了）

❌ 不用：

  • 同一页面 / 区域多个主按钮并排（应仅 1 个主按钮，其余降级为次要 / 链接）
  • 非首要操作（取消 / 返回 / 关闭 → 次要按钮或链接按钮）
  • 列表项中的行内操作（用图标按钮或链接按钮，避免视觉抢戏）

错误示例

  • ❌ 不要硬编码 #FF0F23 替代 token color_primary —— 主色由 V16 Foundation 集中管理
  • ❌ 不要在非首要操作场景使用主要按钮 —— 同页面 / 同区域应仅 1 个主按钮
  • ❌ 不要把 18pt Semibold 改成其他字号字重 —— 按钮文案字号字重是组件契约的一部分
  • ❌ 不要把按钮圆角改成与 radius_l（8px）不同的值
  • ❌ 不要把点击高亮态 overlay #FFFFFF@90% 换成其它透明度

底部导航栏 定义

底部导航栏（Tabbar）位于屏幕底部，作为页面导航的最高层级，管控全局，点击切换整页内容。它分为常规底导和 Joy Agent 组合底导两种形态，支持 2~5 个坑位 + 可选灵动岛运营资源位，并在不同 OS 端 / 版本下选择不同材质（iOS 26+ 液态玻璃 / Android 与 iOS 老系统毛玻璃）。

行为准则

  • 基础切换：点击坑位，切换整页内容
  • 状态切换：默认态 ↔ 选中态（图标 + 文本变 jdred，背景变 gray_6）
  • 灵动岛激活：对应坑位选中态背景消失；同时 Joy Agent 向外抽缩 16 → 28 DP
  • 招手降级：底部存在灵动岛时，文字 / 数字招手自动降级为红点型，灵动岛消失后恢复
  • Joy Agent 招手：气泡缓出向上弹出，最多 18 汉字
  • Joy Agent 点击：进入对话中心场景弹层

底部导航栏 类型

本组件按「形态 × 坑位 × 灵动岛 × 招手」多维度组织。组件层两种主形态：

  • 组件-普通形态（Relay 266:674）：无 Joy Agent，底导宽 351 DP，2~5 坑位均分 + 可选灵动岛
  • 组件-Joy Agent 形态（Relay 266:475）：含左侧 Joy Agent，底导宽 319 DP，Agent 与 Tabbar 间距 8 DP

完整变体矩阵见 variants.md（含 atom 层「12 状态 × 4 招手」组合）。

底部导航栏 结构

Tabbar 在 Relay 中按「原子 → 组件」两层组织。Atom 是不可拆的视觉单位，组件层只做布局组合，不修改 atom 内部规则。

典型场景示意

v0.7 由 outline 自动归纳为 4 类典型场景：

  • A · Joy Agent + 常规底导（无灵动岛 / 无招手）—— 默认状态，占样张大多数
  • B · Joy Agent + 常规灵动岛 —— 日常商品复购提醒（鲜奶定期购 / 订阅续费）
  • C · Joy Agent + 运营 / 大促灵动岛 —— 运营会场推广 / 618 / 11.11 重点大促
  • D · Joy Agent + 招手气泡 —— 新用户 onboarding / 长期未使用提醒

错误示例

  • ❌ iOS 系统安全区 17 DP 内不可放置任何操作
  • ❌ 坑位数禁止设置 6 个及以上，建议 2~5 个
  • ❌ 营销态仅可替换 38×38 图片，不可做其它样式更改
  • ❌ 文字型招手最大 4 字符；Joy Agent 招手气泡最多 18 汉字
  • ❌ 灵动岛红色区域禁止出现任何元素
  • ❌ 运营 / 大促灵动岛必须使用浅色系背景 + 深色字，避免红配绿 / 纯黑 / 纯白
  • ❌ Tabbar 不承担二级页面导航，应由 NavBar / SegmentControl 承担

视频场域底导-单坑原子 定义

视频场域底导单坑原子是逛 / 视频频道在底部导航中的 44×44 slot 内容。它继承通用 tabbar 的容器、坑位数和安全区规则，只补充视频场域自己的单坑视觉：默认态为黑色 video icon + 文案，选中态为淡灰容器 + 红白实心 video icon + 红字 Semibold；icon / 营销态则替换为 38×38 圆形位图插槽。

行为准则

设计师描述手势、转场、状态切换、边界情况。提示：12 COMPONENT 在 root COMPONENT_SET 下，按「底导类型 × 招手类型」二维切换。

  • 默认 → 选中 的转场（颜色 / 缩放 / 反馈）
  • 招手类型切换规则（红点 / 数字 / 文字 / 营销 的触发条件）
  • icon 形态 vs 文字形态的运用约束

视频场域底导-单坑原子 类型

视频场域底导-单坑原子 结构

单坑骨架（44×44 垂直布局）：

  • icon 槽位「透明层」FRAME 20×20（含 #FFFFFF@90% + IMAGE 占位位图）
