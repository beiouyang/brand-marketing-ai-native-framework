SECTION 03 · COMPONENT DIRECTORY

组件目录 · multi-md 文件结构

业务线设计师最常用的形态。每个组件 = 一个目录 = 多个独立 md 文件。每个 md 由对应角色独立 PR/Review,互不打架。点击文件名跳转到 架构大图的 Templates 章节 看具体模板。

# md 文件 谁写 写什么
📄README.md组件 owner概述 + Meta (YAML frontmatter)
📄business.mdPM / 产品业务目标 / KPI / 商业载荷
📄research.md ★用研团队用户画像 / 痛点 / 行为数据 (Why)
📄experience.md ★体验设计师用户旅程 / 决策点 / 家族 DNA / 跨场景同物种 (How)
📄visual.md ★视觉设计师Token 引用 / 变体 / 视觉层级 (What)
📄interaction.md交互设计师状态机 / 动效 / 边界态
📄content.md内容运营标题写法 / 用词 / 语气 / CTA 词
📄donts.md协作收集禁止用法 (视觉/体验/场景禁止)
📄ai-schema.md组件 ownerAI 消费字段 (给 Skill / agent 读)
📄CHANGELOG.md自动生成从 Git commit 自动维护 + 关键变更人工补
📁variants/设计师变体案例库 (子目录)
📁examples/设计师示例资源 / Figma 链接 / 实际截图
SECTION 04 · DIMENSIONS

维度边界 · 用研 / 体验 / 视觉 不打架

这是你最关心的:多角色写一份文档,字段归属如何分? 用研管 Why · 体验管 How · 视觉管 What。

内容字段 用研 (Why) 体验 (How) 视觉 (What)
用户画像 / 痛点★主写引用引用
AB 数据 / 转化★主写引用—
用户旅程位置引用★主写—
决策点 / 规则—★主写引用
跨场景同物种—★主写—
家族 DNA—★主写引用
Token 引用—引用★主写
字号 / 色值 / 圆角——★主写
变体 / 节奏—★主写★主写
禁止用法 (Don'ts)引用★主写★主写
★ 主写: 这个字段由该角色作为主驱动方写; 引用: 该角色在自己的段落里 link 到主写方,不重复内容; —: 该角色不参与这个字段。
SECTION 05 · LINKS

跨 Zone · link 引用规则 (不重写)

DESIGN.md 内部如何引用其他 Zone 资产 —— 永远 link,永远不复制。改源头,所有引用自动生效。
