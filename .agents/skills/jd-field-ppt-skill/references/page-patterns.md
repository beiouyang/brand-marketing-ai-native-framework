# 页面版式登记

正文页优先使用以下 `data-layout`。如确需新增版式，先扩展本文件和校验脚本。

## L01 cover-field

封面。左侧大标题和一句判断，右侧为业务对象图、界面截图或抽象流程，不放装饰图。

## L02 agenda-bands

议程/章节。3-5 条横向带状结构，每条包含编号、章节名、关键问题。

## L03 executive-summary

高层摘要。顶部一句总判断，下方 3 个结论块，每块包含结论、证据、动作。

## L04 kpi-strip

指标页。顶部结论，下方 3-5 个 KPI，底部说明口径和下一步动作。

## L05 compare-before-after

前后对比。左右两栏：现状/目标、旧链路/新链路、问题/方案。中间可用箭头或红色细线连接。

## L06 journey-lane

链路页。横向 4-6 步，适合用户旅程、商家操作、交易路径、设计产出流程。

## L07 matrix-priority

优先级矩阵。2x2 或 3x3，轴必须明确，适合机会评估、风险分层、需求排序。

## L08 anatomy-system

结构拆解。左侧放界面/组件/业务对象，右侧用编号解释结构、状态、约束。

## L09 evidence-table

证据表。适合实验组/对照组、渠道表现、问题清单。行数控制在 4-7 行。

## L10 roadmap

路线图。按时间或阶段展开，必须包含产出、责任边界和验收指标。

## L11 risk-control

风险和兜底。列出风险、触发条件、监控指标、兜底动作。

## L12 closing-action

结尾页。只保留 1 个最终判断和 2-3 个下一步动作。

## 组合建议

- 业务复盘：L01 → L03 → L04 → L09 → L05 → L10 → L12
- 产品方案：L01 → L03 → L06 → L08 → L05 → L11 → L10 → L12
- 设计规范：L01 → L02 → L08 → L05 → L09 → L10 → L12
