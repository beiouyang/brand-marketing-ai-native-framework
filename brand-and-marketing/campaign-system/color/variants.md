# 色彩 · 形态与场景维度

> design.md → [主文档](./design.md) · 同 bundle: [spec](./spec.md) · [behaviors](./behaviors.md)

## 维度总览

| 维度 | 取值 | 说明 |
|---|---|---|
| 规范来源 | 继承 V16.0 色彩规范 | 常规信息表达、状态反馈、基础组件颜色回到基础体系。 |
| 色彩角色 | 活动主题色 / 活动氛围色 | 定义职责和使用关系，不固定色值。 |
| 使用强度 | 轻 / 中 / 重 | 按活动阶段和颜色占比建立使用梯度。 |
| 使用阶段 | 预热期 / 场景期 / 高点期 | 从点缀到主色调再到高识别度强化。 |
| 应用原则 | 活动识别 / 心智匹配 / 主题管理 | 管理活动色彩的一致性、认知匹配和排期冲突。 |

## 色彩角色

### 活动主题色

活动主题色是建立活动识别的主导颜色。每个活动仅存在一个主主题色，用于承载活动核心心智。

适用对象：

- 平台级大促主视觉
- 活动会场主色调
- 核心转化模块强调
- 同类活动长期识别色

### 活动氛围色

活动氛围色围绕活动主题色延展，用于补充页面氛围、层次和情绪表达，不独立承担活动识别。

适用对象：

- 背景氛围
- 辅助色块
- 局部装饰
- 同主题下的色阶延展

## 使用强度

| 强度 | 阶段 | 特征 | 使用倾向 |
|---|---|---|---|
| 轻 | 预热期 | 红色系颜色占比 `≤10%`，点缀使用 | 保持页面信息清爽，降低强转化压力。 |
| 中 | 场景期 | 红色系颜色占比 `≥30%`，主色调使用 | 建立活动氛围，让用户明确进入营销场景。 |
| 重 | 高点期 | 高识别度强化，色彩占比更集中 | 用于活动爆发期、核心利益点和强转化节点。 |

## 组合关系

| 组合 | 说明 |
|---|---|
| 基础色彩规范 + 营销继承说明 | 当前规范不是完整基础色板替代页，而是营销场景的继承与扩展页。 |
| 活动主题色 + 活动氛围色 | 主题色建立识别，氛围色围绕主题色服务表达。 |
| 活动阶段 + 使用强度 | 预热期、场景期、高点期对应轻、中、重的色彩强度。 |
| 活动心智 + 主题管理 | 节庆活动遵循文化心智，相邻活动避免主题色高度重复。 |

## 缺省状态

色彩不因组件默认、按下、禁用、加载等交互状态改变角色定义。交互状态仍应回到基础组件状态色和反馈规则；营销色彩只决定活动识别、氛围透传与页面主色关系。
